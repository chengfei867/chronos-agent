## Pitfalls / learnings

- **GitHub API read via gh-proxy is broken for JSON** — response body comes back as zstd-compressed bytes without a `Content-Encoding` header, so `json.loads` chokes. Fix: read-only `api.github.com/...` calls work direct (no proxy needed); gh-proxy is only required for git push. New rule to propagate into future rounds.
- **PR status `unstable` ≠ blocked** — means "no required checks configured", merge still fine. Only `blocked` / `dirty` would stop admin merge.
- **Previous-round handover gap**: earlier cron slot pushed branch + opened PR #3 but did **not** write progress doc or merge, leaving state ambiguous for the next slot. Two-round handovers like this need at minimum a stub progress doc even if the merge hasn't happened — otherwise the next agent has to re-verify everything from scratch. Will apply going forward (though this case was recoverable in a single slot once diagnosed).
