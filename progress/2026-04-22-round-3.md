## Key Learnings

1. **Migration idempotency is subtle.** Round 3 draft had `INSERT OR REPLACE` in
   `001_init.sql` for the `schema_info` row — this meant a tampered future-version
   row would be silently "fixed" back to `0.1.0`, hiding the incompatibility the
   library is supposed to detect. Switched to `INSERT OR IGNORE` so the seed
   only writes on fresh DBs. Test `test_schema_version_mismatch_raises` caught this.

2. **Fork semantics → SQL semantics.** First draft used `INSERT OR REPLACE` for
   `put_fork`; this silently replaced existing forks when a duplicate `child_run_id`
   was inserted, defeating the UNIQUE constraint. Forks are historical events —
   plain `INSERT` is correct; idempotency belongs in callers via `get_fork_for_child`.

3. **Inline-script subprocess testing is cursed.** Tried generating the writer
   subprocess as a `textwrap.dedent` f-string inside the test fixture — f-string
   double-brace escaping became unmaintainable. Moved to a sibling `fixtures_writer.py`.
   Lesson: **if a subprocess script is >20 lines, put it in its own file.**

4. **UTC is a `datetime` import now (Python 3.11+).** `from datetime import UTC`
   replaces `timezone.utc`. Ruff auto-upgrades this.
