## Why this round

R20 ended with two concrete recommendations and zero code changes:

1. Formalize the ADR-008 "fork=JSON-only" freeze decision now that three
   dogfood rounds back it up.
2. Fix the F2 DX wart (`Node.model_name` vs `Node.usage.model_name`
   cognitive friction — three scripts, three bugs, same mistake).

R21 executes both, cleanly.

## What happened

### R21-A: ADR-013 (5 min)

Wrote `docs/decisions/ADR-013-fork-auto-execution-stay-frozen.md`.

Key choices in the ADR:

- **Status: Accepted**. Not "proposed" — three rounds of evidence is
  sufficient for the bar this decision needs (stop-thinking-about-it,
  not ship-forever).
- **Evidence quality called out honestly**: weak-consistent, single user,
  single LLM backend, single framework. The consistency across
  topologies is what makes it usable despite thinness.
- **Trigger conditions for reopening** listed explicitly (external user
  issue / topology hits hard limit / Phase 2 adapter requires it).
  Keeps the decision reversible.
- **Deferred alternatives** include a **real** middle-ground option (C:
  "emit pastable glue code instead of executing") worth considering in
  R22+. Not lost.

Commit-worthy framing: "this is a stop-wasting-bandwidth decision, not
a forbidden-forever one."

### R21-B: `Node.model` property + docstring cross-refs (8 min)

Three changes in `src/chronos/core/models.py`:

1. `Usage` class docstring gained a "**Note:**" block explaining that
   `model_name` is **not** on this class, with a pointer to
   `Node.model_name` and a self-deprecating reference to R20 Finding #2.
2. `Node.usage` field gained its own inline docstring repeating the
   cross-ref. Pydantic v2 supports field-level docstrings directly
   after the annotation — works as shown.
3. New read-only `Node.model` property returning `self.model_name`.
   Docstring explains the why (three dogfood scripts stumbled, this is
   the shorter canonical form).

No schema change. `Usage` still has only token fields. No cost added to
any code path — property access is ~30ns.

Em-dash check: the first draft of the `Usage` docstring used U+2014.
Ruff's RUF001 rule correctly flagged it; replaced with `--`. This is
the same recurring trap from prior rounds — memorized now.

### R21-B tests: 3 new (2 min)

Added to `tests/unit/test_models.py`:

1. `test_node_model_property_aliases_model_name` — positive path with a
   real model string.
2. `test_node_model_property_none_when_model_name_unset` — null-safety
   on non-LLM nodes.
3. `test_usage_has_no_model_name_field` — **guardrail** for anyone
   tempted to "fix" the DX wart by moving `model_name` onto `Usage`.
   Asserts both `hasattr(u, "model_name") is False` and
   `"model_name" not in Usage.model_fields`. ADR-013 explicitly rejects
   that move; this test bolts the door.

## Release strategy: no bump this round

R21's changes are too small for a version bump.

- Diff size: ~30 LOC of code + ~15 LOC of docstring + 1 ADR file.
- v0.1.4 was cut ~6 hours ago (R19). Cutting v0.1.5 here would make
  the CHANGELOG read like hourly noise.
- `Node.model` is **additive** (new read-only property, no behavior
  change to existing fields). It can go into `[Unreleased]` and
  accumulate with future changes until there's a coherent theme.

Commit lands on main with `[Unreleased]` CHANGELOG entry. Next round
that produces substantive user-visible change will be the v0.1.5 cut.
