## ADR-009 设计决策 (5 选 1)

5 种实现方案对比:

| 方案 | 评价 |
|---|---|
| **A: Callable Protocol hook** | ✅ 被选中 — 显式 API, 失败隔离, 测试友好 |
| B: 全局 callback registry | ❌ 状态散落, 测试要 monkeypatch, 违反"显式优于隐式" |
| C: 让用户继承 `LangGraphRecorder` | ❌ 继承比组合脆弱, 一个子类只能挂一个 extractor |
| D: Middleware chain | ❌ 对单一 concern 过度工程化, 调试难 |
| E: 运行时拦截 LLM 调用 | ❌ 要 monkey-patch 或 import hook, adapter-specific, 碰到自定义 LLM 就崩 |

**最终协议**:
```python
UsageExtractor = Callable[[UsageContext], UsageResult | None]
```
其中:
- `UsageContext` frozen dataclass: `node_name / pre_values / post_values / task`
- `UsageResult` frozen dataclass: `prompt_tokens / completion_tokens / cost_usd_cents / model_name` (都是 optional)
- **失败语义**: 任何 raise → WARNING log + 存 `usage=None`, **永不破 capture** (关键保证)
- 返回 `None` → 等同于不填 (完全合法, 比如 router 节点)

## R12 新增验证事实 (写进 CONTEXT.md §5)

1. **Schema 字段闲置是真实欠款** — `Usage` / `cost_usd_cents` 从 M1.1 就在表里, 到 R12 才有 adapter 写. 教训: schema 字段加了就要立刻配 writer 路径, 否则会静静躺着好几轮没人用.
2. **失败容忍设计最关键** — extractor 是用户代码可能抛任何异常; ADR-009 定死 "raise → log warning + 存 None, 永不破 capture", 这是比签名本身更重要的决策.
3. **`--with-usage` 的性能开销是真实的** — 每 run 多跑一次 `get_nodes_for_run` + SUM, 所以设计成 opt-in flag 而不是默认行为. `runs show` 已经在 fetch nodes, 所以那里是无代价的 auto-show.
4. **Ruff RUF001 (ambiguous minus sign `−` 看起来像 `-` 但不是)** — CLI 的 "Δ (B − A)" 表头用了 en-dash, 打动了 ruff 的 warning. 决定保留 (typography 更好), 加 `# noqa: RUF001` 压制.
5. **frozen dataclass `pytest.raises(Exception)` 不够精确** — ruff B017 要求具体异常; 正解是 `pytest.raises(FrozenInstanceError)` 从 `dataclasses` import.
