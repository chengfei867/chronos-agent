## 3. Decisions / gotchas encountered

### 3.1 mypy strict + re-export requires explicit `__all__` on the *intermediate* module

Initial cut of `linear/__init__.py`'s `from .recorder import AdapterError` blew up under mypy strict with

```
[attr-defined] Module "chronos.adapters.linear.recorder" does not
explicitly export attribute "AdapterError"
```

even though `AdapterError` was in `recorder.py`'s namespace (re-imported from `protocols.py`). Fix: add `__all__` to `recorder.py` listing `AdapterError` + `LinearRecorder` + `LinearRuntime` + `ForkRef` + `RunRef` explicitly.

**New rule added to CONTEXT.md §5 旧事实**: when module A re-imports `X` from `protocols.py` and module B re-exports A's `X`, mypy strict with `implicit_reexport=False` requires A's own `__all__` to list `X` too. The error message is misleading ("does not explicitly export") — the symbol is there, it's just not `__all__`-advertised. Same rule will bite the AutoGen adapter when it lands.

### 3.2 Duck-typed stubs in `TestProtocolConformance`

For `AdapterProtocol` / `NodeIdentityResolver`, no concrete implementation ships in R31. Tests use minimal duck-typed stubs + `isinstance(stub, AdapterProtocol)` to exercise the `@runtime_checkable` decorator — this lets us prove *Python-level* structural conformance today, ahead of R32-B's real `langgraph_adapter` / `linear_adapter` module-level instances.

### 3.3 Context-manager return type in `RecorderProtocol`

Went with `AbstractContextManager[RunRef]` / `AbstractContextManager[ForkRef]` from `contextlib`. This matches the `@contextmanager`-decorated functions on both adapters at the type level and avoids having to bikeshed `ContextManager` vs. `Generator` in the Protocol signature. Conformance tests already pass.

## 4. What this unblocks

- **R32-A AutoGen adapter**: the `from chronos.adapters.protocols import RecorderProtocol` import that the AutoGen adapter will need is live. No more "which module's `RunRef` is canonical?" ambiguity before a third adapter lands.
- **R32-B module-level adapter instances**: can now write `langgraph_adapter = _LangGraphAdapter()` objects that claim `AdapterProtocol` conformance with an `isinstance()` test, not just a hand-wavy "looks right" check.
- **Future adapter registry**: whoever writes `chronos.adapters.registry` later can enumerate adapters by duck-typing against `AdapterProtocol`, not by maintaining a hand-curated list.
- **Eliminates R28 L4 tech-debt ticket** ("two parallel `RunRef` / `ForkRef` / `AdapterError` hierarchies") that was filed against ADR-014 R1's implementation half. Gone before the AutoGen adapter would have added a third.
