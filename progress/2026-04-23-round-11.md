## 关键设计决策 (ADR-008 浓缩)

**核心认知**: `recorder.fork()` 是 context manager, 要求用户代码持有 `graph` 对象并在 `yield` 后 `graph.invoke()`。CLI 是独立进程, 没法凭空构造用户的 graph。

**三个选项**:
1. **Inspection-only** (`fork show`) → 等同已有 `forks show`, 零新价值 ❌
2. **Dynamic import** (`--script mymod.py`) → 安全风险 + 强耦合 ❌
3. **Plan-file** (`fork plan` 生成 JSON artifact, 用户代码 `load_plan()` 消费) ✅

**选 3 的理由**:
- CLI 承担**参数拼装 + fork-point 定位 + override key 校验** (默认拒绝不在 `state_after` 里的 key, `--allow-new-keys` 显式放行 → 捕捉 typo)
- JSON artifact 可 version-control、可贴 bug report、可 diff
- 零动态 import, 零安全风险
- 与现有 `chronos runs show / diff / replay` 美学一致 (CLI 只读 DB, 不执行用户代码)

**Schema v1 的对外契约** (在 CONTEXT.md 硬约束里固定):
```json
{
  "chronos_fork_plan_version": 1,
  "parent_run_id": "...",
  "parent_node_id": "...",
  "parent_node_name": "research",
  "parent_node_index": 1,
  "child_thread_id": "linear-parent-fork-ba418e44",
  "overrides": {"research": "alt-take"},
  "reason": "swap researcher prompt",
  "tags": ["experiment"],
  "task_description": null,
  "generated_at": "2026-04-23T05:04:46+00:00",
  "chronos_version": "0.1.0.dev0"
}
```

## 过程中踩的坑 & 修复

### 1. `recorder_kwargs()` 浅拷贝陷阱
**症状**: 首轮 `test_fork_plan.py` 里 `test_overrides_not_mutated_by_recorder_kwargs` 失败 —— 给 plan 加了 `overrides={"lst": [1, 2]}`, 调 `recorder_kwargs()` 后修改返回的 list, 结果 plan 原始 overrides 也被改。
**原因**: `dict(self.overrides)` 只 copy 外层 dict, 嵌套 list/dict 仍共享。
**修复**: 改 `copy.deepcopy(self.overrides)`。测试从粉变绿。
**教训**: dataclass 暴露可变字段的 accessor 方法, 默认要 deepcopy —— 不是所有调用方都会善待返回值。

### 2. README 里 `## Environment variables` 标题被 patch 吃掉
**症状**: 我用 patch 在 `docs/cli-reference.md` 的 "## Environment variables" 前插入 fork plan 章节, 但 old_string 包含标题那行, 新文档里这一行消失了。
**修复**: 下一个 patch 把标题加回来。
**教训**: patch 前最好先 `read_file` 确认锚点上下文, 宁可 old_string 只含一行不含标题, 也不要"顺手"替换掉标题行。

### 3. CHANGELOG 里 R10 entry 被我覆盖了
**症状**: 第一次改 CHANGELOG 时, 我把 `[Unreleased]` 下的 R10 条目整段替换成 R11。
**发现**: patch 完 diff 显示"Removed (Round 10)"后我立刻意识到不对。
**修复**: 第二个 patch 把 R11 压在 R10 之前, R10 内容完整保留。
**教训**: CHANGELOG 的 `[Unreleased]` 是**栈结构**, 新轮次往上堆, 不是 replace。

### 4. Milestone 编号与 README roadmap 撞车
**症状**: CHANGELOG 里我先给 R11 标了 M1.8, 但 README roadmap 表里 M1.8 已经是 "Structural diff" (R8 ship)。
**修复**: 改成 M1.10 (M1.9 是 v0.1.0 release, M1.10 对应 R11 fork CLI + v0.1.1)。
**教训**: 动里程碑编号前先 grep README/roadmap 确认没占用。

## Artifact 示例

用户运行 `uv run python examples/linear_pipeline.py` 后, dogfood 自动执行并生成:

```bash
chronos fork plan <parent_id> --at-node research \
    --override research="alt" --reason demo \
    --out /tmp/linear_fork_plan.json --db examples/chronos.db
```

产出:

```json
{
  "chronos_fork_plan_version": 1,
  "parent_run_id": "2d8ba237-d4c2-4adc-bb16-78855e5c1140",
  "parent_node_id": "48770a9d-a02e-4556-af12-e60ddc970828",
  "parent_node_name": "research",
  "parent_node_index": 1,
  "child_thread_id": "linear-parent-fork-ba418e44",
  "overrides": {"research": "alt"},
  "reason": "demo",
  "tags": [],
  "task_description": null,
  "generated_at": "2026-04-23T05:04:46+00:00",
  "chronos_version": "0.1.0.dev0"
}
```

用户拿这个 JSON 在代码里:

```python
from chronos.fork_plan import load_plan
plan = load_plan("/tmp/linear_fork_plan.json")
with recorder.fork(graph, **plan.recorder_kwargs()) as ref:
    graph.invoke(None, {"configurable": {"thread_id": plan.child_thread_id}})
print("forked →", ref.child_run_id)
```

4 行完成 fork。
