### M1.1 — Three spikes all PASS ✅

**Spike 1** (`tests/spikes/test_spike1_capture.py`):
- 5-node linear LangGraph: plan → research → draft → review → finalize
- `InMemorySaver` checkpointer attached at compile time
- Verified execution order matches edges; verified each step recorded to `get_state_history()`
- Finding: 5-node graph produces **6 post-input snapshots** (pre-each-node + post-final). Captured in ADR-002.

**Spike 2** (`tests/spikes/test_spike2_fork.py`):
- Ran thread-A to completion; located snapshot with `next == ("draft",)` (i.e., "research just done")
- Copied that snapshot's `values` onto thread-B via `graph.update_state(cfg_b, forked, as_node="research")`
- Mutated `research` field to `"[HIJACKED]..."`; invoked `graph.invoke(None, cfg_b)`
- Verified: thread-B completed; thread-B's `final` ≠ thread-A's `final`; thread-B's `draft` fingerprint ≠ thread-A's.
- This is the **core fork proof**. If this were infeasible, chronos-agent would be dead-in-the-water.

**Spike 3** (`tests/spikes/test_spike3_diff.py`):
- Defined `RunDiff`, `StepDiff`, `FieldDiff` dataclasses
- Wrote `diff_runs(graph, cfg_a, cfg_b, *, ignore_fields=...)` which aligns snapshots by index and reports field-level diffs
- 3 sub-tests: identical runs → empty diff; forked runs → non-empty diff with correct divergent fields; `summary()` is human-readable.
- **Finding**: index-based alignment is **incorrect** for fork-style diffs because `update_state(as_node=...)` offsets thread-B's step numbering. Flagged in ADR-002 as must-fix for M1.7. Spike intentionally stops at the naive algorithm — we wanted to surface this class of issue early, not solve it in a spike.

## Surprises / discoveries

1. **FakeLLM strategy paid off immediately.** All 6 spike tests are deterministic, zero network dependency. This pattern will become the default for integration tests too.
2. **`get_state_history` returns newest-first.** Will bite future implementers. Captured in ADR-002.
3. **`update_state(as_node=X)` lets you "teleport" a new thread into mid-graph position.** This is cleaner than any alternative I considered. LangGraph essentially supports fork natively; Chronos' value is the UX on top (CLI, diff, cost/latency overlay, cross-run comparison).
4. **Semantic alignment is a must for diff.** Index alignment *looks* right for same-thread diffs but falls apart for fork diffs. Need to revisit algorithm in M1.7.
5. **coverage.py warnings about "Module chronos never imported" during spike-only runs** — harmless; spikes don't exercise production code. Full `pytest` run (unit + spikes) gives 98%.
