**Date**: 2026-04-23
**Duration**: ~4.5 hours (includes OneAPI debugging)
**Outcome**: 3 real Chronos bugs found & fixed. ADR-011 written. First case study published.

## What happened

### Phase 1 — target selection

Polled 6 LangGraph-ecosystem repos via GitHub API. Picked
`langgraph-supervisor-py` (8/10):

- Multi-agent pattern → good showcase for per-node attribution
- Officially semi-deprecated → likely real pain points
- OneAPI-compatible (Anthropic Bedrock path)

### Phase 2 — environment

- Default pypi timed out in sandbox. Recurring R17-era workaround:
  `UV_INDEX_URL=https://mirrors.aliyun.com/pypi/simple/`.
- OneAPI model ID resolution was the hardest part of the whole round.
  Final working incantation: `ChatAnthropic(model="Claude Opus 4.7", ...)`
  with **NO `temperature` parameter** (Opus 4.7 is thinking-mode and
  rejects it). Memory updated for future rounds.

### Phase 3 — Bug #1: JSON serialization

First real run crashed on
`TypeError: Object of type HumanMessage is not JSON serializable`.
`_coerce_state` was shallow `dict(values)`; pydantic `BaseMessage`
went straight to `json.dumps`.

Fix: recursive `_jsonable` helper (pydantic → `.model_dump()`,
`__dict__` fallback, primitives pass-through, repr for exotics).
Applied at the 3 `_coerce_state` sites in `adapters/langgraph.py`.

### Phase 4 — Bug #2: silent "no checkpointer" requirement

`LangGraphRecorder` requires checkpointer-backed graphs but doesn't
document it. User sees a LangGraph-native error and assumes config bug.
Noted in docs/case-studies; doc fix deferred.

### Phase 5 — Bug #3: extractor regression from Bug #1 fix

After Bug #1 fix, the run succeeded but every node had `tokens=0+0`.
Root cause: `_jsonable` dict-ifies messages, all three extractors use
`getattr(msg, "usage_metadata")` which returns None on dicts.

Fix: `_msg_field(msg, key)` helper in `langgraph_usage.py` that
dispatches on `isinstance(msg, dict)`. All three extractors updated.

### Phase 6 — regression coverage

Added 6 new unit tests:

- `_jsonable` on pydantic models
- `_jsonable` on nested collections
- `_jsonable` on primitives
- `_jsonable` on exotic types (datetime, UUID, Enum, bytes)
- `aimessage_usage_extractor` on dict-coerced message
- `anthropic_usage_extractor` on dict-coerced message

Total: 236 → 242 tests passing. Coverage 93% → 94%.

### Phase 7 — docs

- `docs/decisions/ADR-011-state-serialization-boundary.md` — decision
  record for the recursive-coercion approach and rejected alternatives.
- `docs/case-studies/langgraph-supervisor.md` — first case study,
  end-to-end story of the dogfood including findings, the working
  trace, and "what we did NOT test".

## Decision about option A (fork-execute)

User asked mid-round whether option A still needed to happen. Answer
encoded here for future rounds:

**A stays deferred.** No usage pattern in this dogfood wanted
auto-execute. What users want first is:
- Native extractors that Just Work (done ✅)
- Cross-run token-delta comparison (future ADR)
- Sub-graph attribution (future ADR)

ADR-008 boundary not breached. R17 produced zero evidence in favor of
lifting it.
