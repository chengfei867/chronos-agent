## 2. R32-B work — module-level `AdapterProtocol` instances

### 2.1 Design

Two module-level singletons, one per shipping adapter, both satisfying `AdapterProtocol` structurally:

| Adapter     | Location                                    | `name`      | `version_constraint` |
|-------------|---------------------------------------------|-------------|----------------------|
| `langgraph_adapter` | `src/chronos/adapters/langgraph.py` (bottom) | `"langgraph"` | `">=1.1,<2"` |
| `linear_adapter`    | `src/chronos/adapters/linear/__init__.py` | `"linear"` | `""` (empty — zero-dep allowed per ADR-016 P2) |

Each is an instance of a tiny private `_LangGraphAdapter` / `_LinearAdapter` class that carries only the two Protocol attrs + a `build_recorder()` method. No mutable state.

### 2.2 `build_recorder()` signatures

Both adapters expose the identical shape:

```python
def build_recorder(
    self,
    store: SqliteStore,
    *,
    kind_map: dict[str, NodeKind] | None = None,
    usage_extractor: UsageExtractor | None = None,
    **adapter_specific: Any,
) -> RecorderProtocol: ...
```

- **LangGraph**: forwards `kind_map` + `usage_extractor` to `LangGraphRecorder.__init__`. Any `**adapter_specific` kwarg → `AdapterError` (LangGraph is a first-class adapter with no framework-specific construction knobs).
- **Linear**: raises `AdapterError` for non-`None` `kind_map` (directs caller to `LinearRuntime(kind_map=...)`) and non-`None` `usage_extractor` (directs caller to the `__chronos_usage__` state-key hint). Accepts `adapter_name` as the one documented `**adapter_specific` kwarg; any other `**adapter_specific` key → `AdapterError`.

## 3. Decisions / notes

### 3.1 Hard-error on unsupported kwargs vs. silent-ignore

Choice for Linear adapter: when `kind_map` or `usage_extractor` is passed non-`None`, **raise `AdapterError` with the real channel**, not silently drop.

Rationale: `kind_map` doesn't live on `LinearRecorder` at all (`LinearRuntime` owns it). `usage_extractor` isn't consulted by `LinearRecorder` (it uses the `__chronos_usage__` state-key hint). A caller passing either kwarg has a misapprehension about where the hook lives; silently dropping would keep them wrong, whereas `AdapterError` surfaces the misapprehension immediately and the error message names the correct channel. This matches the "no silent data loss" principle already established for `UsageExtractor` failures at R15.

**New rule added to CONTEXT.md §5 旧事实**: `AdapterProtocol.build_recorder()` kwarg not applicable to a given adapter → `AdapterError("Use <correct channel>")`, never silent-ignore.

### 3.2 `_LangGraphAdapter` / `_LinearAdapter` private by convention

Underscore-prefixed class names intentionally — the classes are factories for the singletons, not a user-facing API. Callers import the lowercase singleton (`langgraph_adapter`) as documented by ADR-016 P2. Future registry code could instead enumerate via `isinstance(x, AdapterProtocol) for x in vars(chronos.adapters).values()` without needing the private class names.

### 3.3 `TYPE_CHECKING` guard in `linear/__init__.py`

To keep the linear module truly zero-dep at import time, `SqliteStore`, `NodeKind`, `UsageExtractor` are imported under `TYPE_CHECKING`. The typing-only imports only matter for mypy / IDEs; runtime imports stay minimal.

### 3.4 `uv.lock` churn not committed

On first `git status` the working tree had a `uv.lock` change flipping the index URL from `pypi.tuna.tsinghua.edu.cn` → `pypi.org`. Almost certainly caused by an earlier `uv sync` without `UV_INDEX_URL=<aliyun>` set (CONTEXT.md旧事实). Reverted before R31 commit; the chronos-agent stays on the configured aliyun/tsinghua index.
