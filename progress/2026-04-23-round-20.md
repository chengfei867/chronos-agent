# Round 20 — dogfood #3 (`langgraph-bigtool`) + ADR-013 lift-or-stay

**Date**: 2026-04-23 evening CST  
**Version at entry**: v0.1.4 (tagged end of R19)  
**Version at exit**: v0.1.4 (no code change — pure dogfood + ADR round)

## Why this round

Three reasons, in priority order:

1. **Reach 3 independent dogfood rounds for ADR-008 lift-or-stay decision.** R17 (supervisor) and R18 (swarm) provided 2 rounds of zero-demand evidence for execute-fork. One more round gets us to a "weak-consistent-across-topologies" bar that's principled enough to close the question in writing.
2. **Stress-test a topology orthogonal to supervisor + swarm.** bigtool is single-agent with a meta-tool (`retrieve_tools`) and dynamic tool injection — neither centralized-routing nor decentralized-handoff. Third axis of the topology space.
3. **Find what's broken that R17/R18 didn't expose.** Best-case: nothing new. Realistic case: new friction surface. (Spoiler: real case.)

## What happened

### Setup (10 min)

Chose `langgraph-bigtool` over `langmem` / `open_deep_research` for scope. Sandbox in `/workspace/chronos-dogfood/bigtool/`, uv venv, installed `langgraph-bigtool` + `langchain-anthropic`. Used custom **keyword retriever** instead of real embeddings (OneAPI embedding support untested, keeps dogfood focused on Chronos).

### F1: upstream blocker (5 min to patch)

`langgraph-bigtool 0.0.3` calls `tool_node.inject_tool_args(...)`, but LangGraph 1.1.9 made that method private. `AttributeError`, run aborts before Chronos sees anything.

Monkey-patched site-packages `_inject_tool_args`. Run succeeds. Documented in case study as **not-a-Chronos-bug**, goes upstream.

### Dogfood succeeds (3 min to run + inspect)

5 nodes captured, correct answer ("acos(0.5) ≈ 1.0472"), per-node tokens accurate. Topology:

```
[0] agent           kind=llm  tokens=795+99    model=claude-opus-4-7
[1] select_tools    kind=tool
[2] agent           kind=llm  tokens=1333+64   model=claude-opus-4-7
[3] tools           kind=tool
[4] agent           kind=llm  tokens=1417+42   model=claude-opus-4-7
```

Three agent turns booked as three independent nodes (not collapsed). No R18-style under-count.

### F2: the dogfooder fell into his own API trap — three times

First-cut trace output said `model=-` on every LLM node despite the usage extractor clearly returning `model_name='claude-opus-4-7'`. Probe wrapper showed the extractor was fine. DB inspection showed `n.model_name` was populated correctly. Bug was in my own dogfood script:

```python
u = n.usage
model = getattr(u, "model_name", None) or "-"   # wrong — model_name is on n, not u
```

Then I checked R17 and R18 dogfood scripts. **Same mistake, both of them.** Same agent, three different sessions, three identical bugs. That's not a coincidence — it's a real DX wart.

Cognitive model: *"tokens and model are both LLM-specific, they group together."*  
Actual shape: `Node.model_name`, `Node.usage` (tokens only), `Node.cost_usd_cents` as three **parallel** LLM-specific fields.

Fix: R20's own script; R17/R18 scripts left as-is (they're historical dogfood artifacts, not library surface). A durable fix goes in Chronos itself (see ADR-013 section below).

## ADR-013: fork auto-execution — stay frozen

Three rounds of dogfood × zero instances of "I want Chronos to auto-execute this fork" means I have as much evidence as I'll ever get that execute-forks are **not** the critical missing feature. The JSON-only fork path (ADR-008) is sufficient for everything a real user has wanted across three distinct graph topologies.

Formalize this in **ADR-013**. Key points for the doc:

- Evidence quality: **weak**. Three rounds, one user (me), one LLM backend (OneAPI). Bar for "sufficient to decide" is "coherent across topologies", not "statistically robust". Meets that bar.
- Non-goal: not saying execute-forks are forbidden forever. Saying: **we have no evidence they're the next thing to build**, and Phase 2 roadmap priorities should be set from other signal.
- Actionable from this round: the DX friction in F2 is what we actually learned. **Dock one round of polish work to address F2**, defer execute-fork until someone asks.

## ADR-013 action items

- **Low-cost**: improve docstrings on `Node.usage` and `Usage` to cross-reference `Node.model_name`.
- **Medium-cost**: add `Node.model` read-only property (alias for `.model_name`), less typing, consistent shorthand.
- **No schema change**: `Usage` stays token-only. Moving `model_name` onto `Usage` was considered and rejected (schema break with no upside — the bug is cognitive, not structural).

Deferred to a future round (this round is observational + decision, not fix).

## Lessons

1. **When your own three different scripts make the same mistake, the library is wrong, not you.** Accepting this took one more debug cycle than it should have.
2. **Observational rounds are legitimate.** The urge to "produce code this round" is strong; resisting it when the real learning is "our thesis holds + here's new friction" is valuable.
3. **Third-data-point threshold works.** One dogfood feels like a demo, two like a pattern, three like evidence. ADR-008 can now be lifted-or-stayed on a defensible basis instead of indefinite deferral.
