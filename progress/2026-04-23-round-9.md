### 2. Real bug found: CLI usage wrong in 3 docstrings

Ran `uv run python examples/linear_pipeline.py` (R8's flagship demo)
and tried to run the commands it suggests you copy-paste. They worked.
Then I tried the docstring at the top of the file, which said:

```
chronos --db examples/chronos.db runs list
```

That form **doesn't work** — `--db` is a *per-subcommand* flag, not
a global one, so Typer emits `No such option: --db`. The correct form
is `chronos runs list --db examples/chronos.db`.

Interestingly R8 was half-aware of this — the runtime `print()` output
and `docs/getting-started.md`'s Troubleshooting section both already
had the correct form. R8 just forgot to update the docstrings (which
developers read more than the Troubleshooting section).

Scope of fix:

- `examples/linear_pipeline.py` docstring
- `examples/router_loop.py` docstring
- `examples/__init__.py` docstring

Verified via `grep -r "chronos --db"` after the fix: only remaining
hit is the Troubleshooting paragraph *explaining* the mistake, which
is correct as-is.

## Engineering lessons

### The "progress doc every round" rule is non-optional

R8 skipped it → R9 had to do archaeology on `git diff` to reconstruct
what happened. If R8 had written even 2 paragraphs of "here's what I
did" before hitting the tool limit, R9 could have started a new
direction instead of spending the first third of the round on audit.

Rule reinforced: **write progress doc before the last commit**, so if
the round gets killed, at least the summary is in-tree.

### Dogfooding finds the "obvious" bugs

The `chronos --db X cmd` → `chronos cmd --db X` error is the *kind of
thing* that looks right when you skim but breaks when you actually
copy-paste. R8 wrote the Troubleshooting note ("here's this error,
here's the fix") but didn't scrub the docstrings that *cause* it. No
unit test catches this (it's a human-facing docstring bug). Actually
running `uv run python examples/linear_pipeline.py` then copy-pasting
the first suggested command is what found it.

**Recommendation for R10+**: add a `make dogfood` target (or equivalent
in `justfile` / `Makefile`) that runs both examples, captures their
suggested commands via regex, and executes each one to confirm they
return exit code 0.
