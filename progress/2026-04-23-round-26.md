### 2. ADR-016: Adapter Interface

Drafted `docs/decisions/ADR-016-adapter-interface.md` (~15 KB, Accepted). Three Protocols:

#### `RecorderProtocol` — the core contract

```python
class RecorderProtocol(Protocol):
    def record(self, runtime, *, thread_id, task_description=None, tags=None) -> ContextManager[RunRef]: ...
    def fork(self, runtime, *, parent_run_id, at_node_id, overrides=None,
             child_thread_id, reason=None, task_description=None, tags=None) -> ContextManager[ForkRef]: ...
```

Five lifecycle invariants:
1. **Atomicity** — single `store.transaction()`, no partial runs
2. **Idempotency** — re-entering with same `thread_id` creates a new Run (UUID is the key, not thread_id)
3. **Single failure mode** — `AdapterError` is the only legal framework-leak
4. **Silent no-op** — empty runs (user never invoked) don't create Run rows
5. **Failed runs persist** — `status=FAILED`, populate ref, then re-raise

Rename `graph=` → `runtime=` at the contract level; `LangGraphRecorder` keeps `graph=` as positional-compatible alias (zero breaking changes).

#### `AdapterProtocol` — the plugin shape

```python
class AdapterProtocol(Protocol):
    def build_recorder(self, store, *, kind_map=None, usage_extractor=None,
                       **adapter_specific) -> RecorderProtocol: ...
    name: str
    version_constraint: str
```

`**adapter_specific` is the pressure-release valve — LangGraph needs zero extra kwargs today; AutoGen will likely need `group_chat_filter` or similar.

#### `NodeIdentityResolver` — the Phase-2 hook

```python
class NodeIdentityResolver(Protocol):
    def resolve(self, event: Any) -> tuple[str, NodeKind] | None: ...
```

Why exposed now despite YAGNI: R23 taught us `node_name` derivation is the *first* thing that breaks at a framework boundary. Documenting the hook now (3 lines) > retrofitting later.

#### Rejected alternatives (5)

1. `abc.ABC` — `LangGraphRecorder` is already structurally conformant; ABCs force runtime import dependency
2. Merged `Adapter` Protocol covering both module + instance shapes — mixes import-time metadata with instance behaviour
3. Drop `NodeIdentityResolver` — R23 evidence says the hook is needed, 3 lines is cheap insurance
4. Fold `UsageExtractor` into `RecorderProtocol` — ADR-015 owns `UsageExtractor`; composition over aggregation
5. `runtime: Runtime` (another Protocol) — would bake framework-specific methods into the wall we're trying to build

## Decisions captured

- **ADR-016** owns the adapter interface; "ADR-005 adapter interface" references in roadmap were stale (ADR-005 is fork semantics)
- **`runtime=` is the canonical parameter name** at the contract level; `graph=` remains positional alias in LangGraphRecorder for zero-breakage
- **`AdapterError` is the only legal framework-exception leak** — all other framework exceptions must be caught and wrapped or passed through recorder internals
- **Linear-pipeline adapter is an acceptable R1 implementation** — doesn't need to be AutoGen; ADR-014 R1 says "non-LangGraph", not "AutoGen specifically". Zero external dependencies keeps the ADR-014 red line clean
- **Roadmap drift self-check**: every round end, glance roadmap.md; refresh if >5 rounds stale

## Lessons

1. **Roadmap is not write-once.** R7 was the last refresh before R26. 18 rounds of silent drift turned roadmap into a liability — claiming Phase 1 incomplete while shipping v0.1.6. ADR-014's whole point was preventing this class of doc/reality divergence; R26 is the first round where we *actually enforced* the enforcement ADR.

2. **Protocol choice (Python structural typing) > ABC.** ADR-015 chose Protocol; ADR-016 consistent. `LangGraphRecorder` is a conformant `RecorderProtocol` today without changing a single import — that's the test that structural typing wasn't a mistake.

3. **Speculative hooks (like `NodeIdentityResolver`) earn their keep IF cheap and IF evidence-backed.** 3-line Protocol for something R23 evidence says will definitely matter in Phase 2 = yes. A full "Event taxonomy Protocol" with 5 methods would have failed the YAGNI test.

4. **`**adapter_specific` keyword-only is the only honest pressure-release valve.** LangGraph doesn't need it today; AutoGen will. Forcing a fixed-shape kwarg dict would either over-promise now or force contract churn later.

5. **Renames at the contract level don't need to break existing call sites.** `runtime=` at Protocol, `graph=` positional alias in `LangGraphRecorder` — 3 lines of backwards-compat cost, zero user-facing breakage.
