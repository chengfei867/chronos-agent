# Round 14 ‚Äî CLI file split (2026-04-23)

**Status**: ‚úÖ ship-ready; all quality gates green; no behaviour change

## Goal

`cli/__init__.py` had grown to 848 lines ‚Äî overdue for split (deferred 3 rounds
in R11/R12/R13). R14 extracts each subcommand group to its own module while
preserving the exact user-facing CLI surface.

## Why now (per CONTEXT ¬ß6 R14 candidates)

- Top recommendation: CLI split (overdue, 1-round scope, no ADR needed)
- 848 lines exceeds the "open a file and navigate" threshold; subcommands had
  started accreting helpers (`_RunUsageSummary`, `_render_diff_table`,
  `_render_usage_compare`) that belong to specific commands
- `replay.py` / `fork.py` already established the **implementation-module
  pattern** (thin typer wrapper in `__init__.py` ‚Üí `*_command(...)` in the
  submodule, with `console` and `open_store_fn` injected); applying the same
  pattern to `runs` / `forks` / `diff` closes the inconsistency

### Slimmed entry point
- **`src/chronos/cli/__init__.py`**: **848 ‚Üí 348 lines (-59%)**
- Now contains only: module docstring, typer app objects
  (`app`/`runs_app`/`forks_app`/`fork_app`), `add_typer` mounts,
  `_version_callback`, `main` callback, `info` command, and thin
  `@X.command()` wrappers that import + delegate to `*_command(...)`

## Architectural pattern consolidated

All subcommand implementation modules now follow the same shape:

```python
def <group>_<verb>_command(
    *,
    # command-specific kwargs
    open_store_fn: Callable[[Path | None], SqliteStore],
    console: Console,
) -> None:
    ...
```

Injecting `open_store_fn` and `console` lets tests swap them without
monkey-patching module globals. Matches the pre-existing `replay_command`
and `fork_plan_command` shape ‚Äî no new pattern invented.

## Trade-offs considered

- **Alternative A (rejected)**: Move typer apps to `cli/_apps.py`, have every
  subcommand module register its own `@X.command()` via import side-effects.
  Cleaner in theory (no wrapper plumbing) but breaks the existing
  `replay.py` / `fork.py` mold and makes DI harder for tests.
- **Chosen**: Thin wrappers in `__init__.py`. Keeps consistency with the
  pattern that already survived R10‚ÄìR13 without issue. Wrappers are
  mechanically trivial (~12 lines each) ‚Äî cost accepted for uniformity.

## Lessons / pitfalls noted

1. **First draft of `runs.py` had a typo** ‚Äî I merged the `_RunUsageSummary`
   and `_sum_usage` imports into a single block but accidentally included a
   `_subject_summary := None` walrus placeholder. Ruff caught it instantly.
   Patched immediately. Lesson: when consolidating imports across modules,
   write the target import block in one go, don't leave placeholders.
2. **Ruff's `ruff check --fix`** handled the expected cleanups silently
   (import ordering, `typing.Callable` ‚Üí `collections.abc.Callable`). Always
   run fix + format before mypy/pytest on freshly-written modules.
3. **`uv.lock` drift** did NOT trigger this round (no dep changes). The
   hard-constraint reset dance wasn't needed ‚Äî but stay vigilant for R15.
