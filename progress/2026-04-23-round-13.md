## R14 推荐

CONTEXT §6 会改成:

### 选项 A (R14 强推荐): CLI 文件拆分 — tech debt 还清
`src/chronos/cli/__init__.py` 现在约 **750 行** (R12 又加了 usage helpers + 3 处 flag). 按 subcommand group 拆:
- `cli/__init__.py` — 只留 `app` + `info` + `_version_callback` + 通用 helpers
- `cli/runs.py` — `runs list / runs show`
- `cli/diff.py` — `diff` + `_render_usage_compare`
- `cli/replay.py` — `replay`
- `cli/forks.py` — `forks show`
- `cli/fork.py` — `fork plan`
- `cli/_usage.py` — `_RunUsageSummary` + `_sum_usage` + 4 个 fmt helpers (被 runs + diff 共用)

**优势**:
- 无需 ADR (纯 refactor, 不改对外 API / CLI 行为 / JSON schema)
- 分散后每文件 <150 行, mypy / ruff 更快
- 测试不用动 (`CliRunner` 对模块路径不敏感)

**风险**:
- Diff 巨大, review 负担; 一轮专攻不混功能
- `app.add_typer(runs_app, ...)` 方式如果有 typer 版本兼容坑要注意

**预计**: 1 轮可完成, 不影响对外行为.

### 选项 B: `aimessage_usage_extractor` 扩展到 Anthropic / OpenAI 原生格式
R12 只实现了 LangChain 标准化后的 `AIMessage.usage_metadata`. 如果用户直接调 Anthropic SDK (`usage.input_tokens / usage.output_tokens`) 或 OpenAI (`usage.prompt_tokens / usage.completion_tokens`), 需要额外 extractor. 可新增 `anthropic_usage_extractor` / `openai_usage_extractor` + 短 ADR (接受新输入格式).

### 选项 C: AutoGen adapter (仍需用户点头)
- Phase 2 启动, 硬红线, 除非用户显式同意

### 选项 D: Web UI (不建议 cron 启)

**R14 倾向 A**, 纯技术债还清, R12/R11 连续推迟了, 不还越积越多.
