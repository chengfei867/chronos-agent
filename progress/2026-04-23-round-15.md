## What I decided

Write **two** new convenience extractors as siblings to the existing one,
sharing a small internal helper for the "walk messages from tail, find
first with response_metadata[key]" loop. All three implement the existing
`UsageExtractor` Protocol from ADR-009. Zero protocol change, zero schema
change.

Critical fidelity concerns addressed:

1. **Anthropic prompt caching** тАФ `cache_creation_input_tokens` and
   `cache_read_input_tokens` come out of the API as **separate fields
   from `input_tokens`**. Na├пve extractors forget to add them and
   under-report prompt usage by 10-100├Ч. We fold both into `prompt_tokens`
   on the way into `UsageResult`. The cost differential (cache-create
   +25%, cache-read тИТ90%) stays the user's pricing-table concern per
   ADR-009's "Chronos stays out of the model-pricing business" rule.

2. **OpenAI reasoning tokens** (o1/o3) тАФ the API reports
   `completion_tokens_details.reasoning_tokens` as a **sub-detail** of
   `completion_tokens` (completion already includes reasoning). We surface
   `reasoning_tokens` alongside without subtracting, preserving the
   invariant `prompt + completion == total_tokens`.

3. **Duck typing** тАФ no hard dependency on `anthropic` or `openai` SDKs
   (they aren't even installed in the dev venv). Extractors read field
   names from plain dicts; users without those packages can still use the
   extractors.

4. **Cross-provider fallback** тАФ documented and tested the
   `anthropic or openai or aimessage` composition pattern for users who
   mix providers across nodes.

## What I shipped

## Not shipped (deliberately)

- **Automatic cascade in `LangGraphRecorder`** тАФ rejected in ADR-010 ┬зB.
  Users opt into the exact path that matches their setup; cascade is a
  user-side composition.
- **Pricing tables / cost calculation** тАФ explicitly user turf per
  ADR-009. `cost_usd_cents` stays `None` in all three shipped extractors.
- **`anthropic` / `openai` SDK type hints** тАФ would pull heavy optional
  deps for pure docstring value. Duck typing reads the same field names
  with no install cost.
- **Version bump** тАФ R15 is a pure additive feature on top of the v0.1.2
  hook surface. Bundling it with the cache-semantics refinement into a
  future v0.2.0 (or v0.1.3 patch) feels cleaner than tagging every small
  extractor addition.
