### 2. `tests/conftest.py` (new file)

Autouse fixture `_strip_color_env`. Two-layer fix:

1. **Env layer**: `monkeypatch.delenv` for `FORCE_COLOR`, `NO_COLOR`,
   `CLICOLOR`, `CLICOLOR_FORCE`, `PY_COLORS`. Plus `setenv("TERM",
   "dumb")` and `setenv("COLUMNS", "80")` for deterministic rich
   rendering.
2. **Already-constructed Console layer**: `rich.Console()` reads env
   at construction time, but `chronos.cli._common.console` is
   instantiated at **import** — before any pytest fixture runs. Just
   clearing env is too late for that instance. Fix: `monkeypatch.setattr`
   the module attribute on both `chronos.cli._common` **and**
   `chronos.cli` (the latter does `from chronos.cli._common import
   console`, creating a second binding in its namespace) to a freshly
   built `Console(force_terminal=False, no_color=True, color_system=
   None, width=80, highlight=False)`.

Verification: `FORCE_COLOR=1 uv run pytest tests/ -q` → **264 passed
in 4.75s** (was 3 or 5 failures depending on the day). Without
`FORCE_COLOR`, still 264/264 (fixture is a no-op in that path).

## Findings (new)

### F1. rich's `Console` captures env at construction, not on each print

Confirmed by debugger-less inference: the fixture's `monkeypatch.delenv`
alone didn't fix `test_cli_fork_plan_json_to_stdout` — only adding the
`monkeypatch.setattr` on the module-level `console` objects did. Means
any future test utility that constructs its own Console for
stdout-assert work will ALSO need to pass `no_color=True, color_system=
None` explicitly — env is not a reliable control surface post-import.

### F2. "from X import y" creates a second binding, not a shared reference

`chronos.cli.__init__` does `from chronos.cli._common import console`.
This copies the reference into the `chronos.cli` module namespace.
`monkeypatch.setattr` on one does NOT affect the other. This is
Python 101 but worth remembering — the fix required two `setattr`
calls, not one. Documented in the conftest docstring so future test
authors don't repeat the debugging.

## Lessons for next rounds

- **ADR-only rounds are legitimate.** Project discipline is not always
  code. The R10 near-miss was a process failure that code can't fix;
  ADR-014 fixes it at the process layer.
- **Pair ADR work with one small code deliverable.** Keeps the round
  from being "only a doc." The FORCE_COLOR fix here cost ~20 minutes
  and closed a known report Finding, good ROI for the page-weight.
- **R2 before R1.** When two criteria depend on each other, list the
  substrate first. Extractor contract is substrate for adapter
  interface.
- **Rich Console state is import-time**, so test-time manipulation
  needs `setattr` not just env.
