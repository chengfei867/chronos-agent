## What R10 did

### 1. `scripts/dogfood.sh` + CI wiring (R8→R9 bug-class armour)

Bash script that:
1. Runs every `examples/*.py` through `uv run python`.
2. Extracts the "Try these commands:" block each example prints.
3. Re-executes every suggested command; any non-zero exit fails the
   round.
4. Greps example docstrings for the `chronos --db X cmd` pattern that
   broke in R8 and fails loudly if found.

Wired into `.github/workflows/ci.yml` as a step gated on
`matrix.python-version == '3.11'` (saves time on the 3.12 leg).

**Result**: 12/12 suggested commands green (was 10/10 before we added
the replay commands to examples).

### 2. ADR-007 — replay TUI framework selection

Before writing a line of replay code I forced myself to write the ADR.
Considered `textual`, `prompt_toolkit`, `curses`, pager-style, and
`rich.live`. Picked **`rich.live`**:

- zero new deps (`rich` already pinned)
- aligns with existing `rich.table` usage in `runs show`/`diff`
- simpler mental model than textual's app lifecycle
- for a feature that's already optional (non-TTY falls back to static
  render) the weight of a real TUI framework isn't justified

Full reasoning in `docs/decisions/ADR-007-replay-tui-framework.md`
(6.5KB).

### 3. `chronos replay <run_id>` (M1.7)

Lives in new module `src/chronos/cli/replay.py` (13.5KB). Kept the CLI
package split deliberate — `cli/__init__.py` stays the Typer dispatcher,
`cli/replay.py` owns the rendering + key loop. Did **not** split the
other subcommands out yet; risk-stacking (R8→R9 lesson).

Key pieces:

- `build_frame(nodes, cursor)` — pure renderer that returns a rich
  Panel. Testable without a terminal.
- `advance(cursor, n, direction)` — pure cursor math. Testable.
- `_read_key()` — stdlib raw-tty reader, posix (`termios`+`tty`) and
  windows (`msvcrt.getwch()`) branches. Parses arrow-key escape
  sequences and home/end.
- `replay` Typer command — wires `rich.live.Live` + the key loop,
  auto-falls-back to static output when stdin/stdout isn't a TTY, and
  honours explicit `--no-interactive`.

## Lessons for R11+

1. **Re-read `docs/CONTEXT.md` every round before touching `deps`.**
   R10 started wrong by pulling AutoGen; CONTEXT.md caught it but only
   because I re-read it mid-flight. Make that step first, not fourth.
2. **ADR-before-code works.** Writing ADR-007 first killed at least one
   hour of "should I try textual?" churn during implementation.
3. **Dogfood-as-CI is leverage.** Writing the script took 30 min; it
   will prevent every future R8-style docstring drift for free.
4. **Risk-stacking is real.** I wanted to split `cli/__init__.py` *and*
   ship replay *and* write ADR-007 *and* the dogfood script. Shipping
   replay + dogfood + ADR without the file split was the right call.
