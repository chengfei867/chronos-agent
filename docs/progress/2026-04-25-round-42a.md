## What happened

After v0.2.1 shipped (R41), I moved into Phase 3 entry. The first milestone
on the roadmap — *"side-effectful tool sandboxing (ADR-006; use E2B or
Modal)"* — smelled like drift: ADR-013 (R21) froze fork auto-execution, and
`multi-framework-risks.md` R-6 already argued the current `fork plan` UX is
correct *because* Chronos is out of the execute path. If Chronos doesn't
execute forks, what exactly needs sandboxing?

Rather than write the research note from armchair, I built Spike 8 to
empirically check the core assumption: *when a user forks a LangGraph run
and resumes with `invoke(None, cfg_child)`, do pre-fork side effects get
replayed?*

## Spike 8 results

3-node graph (`plan` → `post` → `final`), `post` does `httpx.post` through
`MockTransport` with a call counter.

| Phase | Scenario | httpx count (expected) | Result |
|-------|----------|-------------------------|--------|
| 1 | Initial recorded run | 1 (1) | ✅ |
| 2 | Fork AT `post` with `post_result` override | 0 (0) | ✅ |
| 3 | Fork AT `plan` with `plan` override (upstream) | 1 (1) | ✅ |

Key finding: **LangGraph's checkpointer already gives us "pre-fork
side-effect immunity" for free.** Forking after a node does not replay it.
Forking before a node re-executes downstream — which is the whole point
and is user-intentional.

## What this changes

The Phase 3 entry point flips from *"integrate E2B or Modal"* (10–20
rounds of sandbox infra) to *"write the side-effect patterns doc + add
effect-kind node metadata"* (3.5–4.5 rounds).

Proposed Phase 3 on-ramp:

- **PH3-01** — `docs/guides/side-effects.md` with 3 patterns:
  MockTransport, envvar kill-switch, pure/effectful tool split. 1 round.
- **PH3-02** — `nodes.effects` metadata + UI warning badge on ForkPlan:
  *"re-executes 2 network-effectful nodes"*. 2–3 rounds.
- **Parking lot** — E2B/Modal sandbox integration, reopens only if
  ADR-013 triggers fire (none active as of R41).

I also annotated the roadmap bullet with a "under R42-A review" flag
pointing at the research note; **did not rewrite it** — roadmap charter
changes belong to the user, not the agent.
