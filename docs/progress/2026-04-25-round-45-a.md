## Deliverables

### 1. Pure helper — `src/chronos/cli/fork.py::build_effects_summary`

```python
def build_effects_summary(
    downstream_nodes: Sequence[Node],
    *,
    dangerous: frozenset[str] = DANGEROUS_EFFECTS_DEFAULT,
) -> dict[str, Any]:
```

Returns `{total, dangerous_count, tag_counts, samples}`:
- `total` — count of downstream nodes (step_index > parent.step_index)
- `dangerous_count` — subset that carry any tag in `dangerous`
- `tag_counts` — `{tag: int}` across all downstream nodes (sorted alphabetically on render)
- `samples` — up to 3 dangerous nodes, each `{step, name, tags}` (prevents 50-node dangerous runs from flooding the terminal)

**Defensive**: if `metadata["effects"]` is missing or not a list (malformed runs), the node is treated as tag-less rather than crashing. No I/O, no console — trivial to unit-test.

### 2. Renderer — `render_effects_preview(summary, console)`

Rich `Panel(border_style="yellow")` titled *"Downstream side-effects preview"*:

```
⚠ 4 dangerous / 5 total
breakdown: db=1, external=1, fs=1, network=2
• step 1: fetch_api (network, llm)
• step 2: write_file (fs)
• step 3: db_insert (db)
… and 1 more
Chronos does not sandbox forks — review before running (ADR-019).
```

**Silence principle**: renders nothing when `total == 0` (forking at the last node) OR `dangerous_count == 0` (pure-LLM downstream). Prevents false-alarm fatigue.

### 3. CLI wiring — `render_plan_preview` + `fork_plan_command`

- `render_plan_preview()` gains `effects_summary: dict | None = None` kwarg (backward-compatible — existing callers pass nothing and get the old behavior).
- `fork_plan_command()` after parent resolution:
  ```python
  downstream = [n for n in nodes if n.step_index > parent_node.step_index]
  effects_summary = build_effects_summary(downstream)
  ```
  Both emit paths (JSON stdout + Python-literal stdout) pass it to `render_plan_preview`.

### 4. Tests — 8 new in `tests/unit/test_fork_cli.py`

Helper block (pure-function tests, no Click runner):
- empty downstream → `{total:0, dangerous_count:0, tag_counts:{}, samples:[]}`

## Key facts (pinned to CONTEXT.md §5)

1. **Per-run linear downstream, not DAG-topological**: `step_index > parent.step_index` matches the replay/fork consumer mental model (forking replays the timeline from that point forward). Walking the DAG via `parent_node_id` edges is a Phase 4 concern and would change semantics.
2. **Silent-by-default**: no panel at `total == 0` or `dangerous_count == 0`. ADR-019's honesty is "warn when it matters", not "spam on every fork".
3. **Sample cap 3 + `… and N more`**: realistic ergonomic cap; enough to give users a feel for what's downstream without flooding a terminal.
4. **Defensive metadata parsing**: `metadata["effects"]` treated as optional-and-possibly-garbage input. One-line `isinstance(x, list)` guard, no recovery magic.
5. **Pure helper / renderer split**: `build_effects_summary()` is easily unit-tested (no I/O, no console); `render_effects_preview()` is a thin renderer smoke-tested via CLI key-string assertions. Good pattern for future CLI additions.
