## 4. 决策 (R60 可上墙的, 供未来 rounds 继承)

### 4.1 "三轮打一 tag" bundle 合法性 (R60 确认, 历史模式继承)

R58 pure core + R59 CLI/API + R60 dogfood + release 的三轮结构 = **一个 Arc slice 的完整闭环**. v0.5.0 bundle 理由:
- R58 没有 user-facing surface, **不宜单独 tag** (无人能用).
- R59 落了 CLI + HTTP, 但缺乏 real-agent 验证.
- R60 dogfood 证明端到端, 加 release cut 让 user 一次收到完整 story.
- 对比 alternative (R58→v0.4.1, R59→v0.4.2, R60→v0.5.0): 噪音多, Arc A 被切碎, release notes 重复.

记法: "Arc slice = core + surface + proof = 1 bundle = 1 minor version". R60 是第 **1** 次显式三轮 bundle (之前 R44-A / R45-A / R47 是单轮 cut). 升级为 soft pattern, 下一 Arc (B 或 C) 继续验证后 promote skill.

### 4.2 Dogfood script 作为 "living design doc" (R60 新)

`scripts/dogfood_compare_n.py` 不仅是 release evidence, 也是 design-doc `§5.1 / §6` 的可执行 spec. Non-maintainer reader 读脚本能 5 分钟理解:
- 怎么构造 pivot + N-1 forks.
- `absent` / `changed` / `added` / `removed` 各自语义.
- JSON schema 长啥样.

**对未来 Arc (B: multi-pivot, C: Web UI)**: 都 budget 一个 `scripts/dogfood_*.py`, 不仅 unit test. Dogfood script 是 "**纸面 design doc → 真 agent trace 打通**" 的桥, 也是 regression guard (任何 signature drift 会在 dogfood smoke 里炸).

### 4.3 `ruff format --check` 只扫 `src/ + tests/` (R60 确认, 非 R60 新)

`scripts/` 是 seed/evidence/one-shot, 风格 drift 不值 regression. Lint (正确性, e.g. `E402` 已 per-file-ignore, `F401` 会扫) 保留. Format (纯风格) 豁免. 记进 CONTEXT invariant.

### 4.4 N=2 frozen-contract 三连守卫在 dogfood 上的再验证 (R59 候选 skill 第二次验证)

R59 列出 "pure + CLI + HTTP 三层 N=2-equals-legacy 测试" 作为候选 skill (`tests/unit/test_merge_pivot_reports.py::test_summary_matches_diff_report_for_n2`, `test_cli_compare.py::test_compare_n2_summary_matches_chronos_diff_summary`, `test_api_server.py::test_compare_n_n2_matches_compare_2run_summary`). R60 dogfood 跑 twin (N=2 + pivot, 即 2-run compare) 的 equal row 是 **6/0/0/0**, 结构上等于 `chronos diff pivot twin` 该输出. 虽没加第 4 层 dogfood 断言 (dogfood 是 evidence 不是 pytest), 但 **观察到 consistency**. 下次 Arc 再验证一次 promote `n2-frozen-contract-triangulation` skill.
