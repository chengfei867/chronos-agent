| D7 | `usage_extractor` callback unsupported, raises `AdapterError` | `__init__.py::_CrewAIAdapter.build_recorder` — `if usage_extractor is not None: raise AdapterError(...)` | ✅ (covered by `TestAdapterFactory.test_usage_extractor_rejected`) |

No deviations. The inherited scaffold is a faithful implementation of ADR-021. The effects classifier (R44-A) is wired through `_persist_run_and_nodes` — `classify_effects(node_name=..., kind=..., model_name=...)` — so a future `spike13` real-LLM run will get effect tags on TOOL nodes for free.

### Minor ergonomic beats worth noting

- **`record_from_pending()` replaced by `_drain_buffer_to_store()` test hook** — the inherited scaffold gives tests a direct `_drain_buffer_to_store(ref=..., ...)` hook that reads buffer under lock and clears it, so tests can exercise handlers without running a real event bus. `TestDrain` uses it extensively. Name is underscore-private so it doesn't bloat the public surface.
- **`_FakeEventBus` in `test_adapter_crewai.py`** — a ~100-line handcrafted fake that mirrors CrewAI 0.80's `scoped_handlers()` + `on(event_type)(handler)` + `flush(timeout=...)` contract closely enough that the record CM can be driven end-to-end. Proves the CM lifecycle (attach → yield → flush-on-success → drain → detach) on a non-CrewAI environment. This is how we end up green without the real `crewai` dep.
- **Optional `CrewKickoffCompletedEvent` import** — ADR-021 §D4 lists it in the default kind map, but CrewAI has moved that class across subpackages across minor versions. The recorder wraps the import in its own `try/except ImportError` and only subscribes `_on_end_event` if the class is available. Failure mode: `CrewKickoffCompletedEvent`-less installs get a run with no explicit END node, but all other events still record. This matches ADR-021's "make events optional at the right granularity" posture.
- **`_persist_failed_shell()`** — writes a minimal failed Run when the `with` block body raises, so partial runs aren't orphaned. Same pattern as AutoGen's failure path.
