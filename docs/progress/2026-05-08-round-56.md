## 7. Load-bearing findings (→ CONTEXT §5 sync)

- **R56 R0-arc pattern** — post-release polish round is canonically write-only: docs + ADR skeleton + progress. Avoid touching `src/` entirely to keep the release-commit and the polish-commit narratives clean.
- **ADR filename verification pre-commit** — always grep the actual `ls docs/decisions/ADR-*.md` output against typed filenames before committing; R56 caught one `ADR-022-crewai-version-pin.md` → `...-pin-bump.md` typo that would have shipped otherwise.
- **Broken-link programmatic sweep** is cheap (20ms Python one-shot) and should be run on any round that touches 2+ .md files with cross-references.
- **`×` avoidance in markdown is a cheap-to-adopt habit** even though technically unblocked — keeps future ruff-discipline low-friction.
- **Phase transition rounds** (like R56) are structurally different from in-phase rounds: they close the prior phase in roadmap, frame the next phase in an ADR skeleton, refresh external-facing docs. The pattern is now documented enough to be a skill candidate (see §8).
