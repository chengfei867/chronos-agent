## §3 The blocker (Option B documentation)

### §3.1 Symptom reproduction

```python
import os, asyncio
os.environ["ANTHROPIC_BASE_URL"] = "<baidu-int relay URL from /workspace/.hermes/.env>"
os.environ["ANTHROPIC_API_KEY"] = "<baidu-int key>"
from claude_agent_sdk import query, ClaudeAgentOptions
async def go():
    async for msg in query(prompt="ping", options=ClaudeAgentOptions(max_turns=1)):
        print(type(msg).__name__, getattr(msg, "model", None), getattr(msg, "error", None))
asyncio.run(go())
```

**Output**: `AssistantMessage <synthetic> authentication_failed` then the SDK hangs on subsequent calls (the `ClaudeSDKClient` session protocol expects ongoing CLI cooperation).

### §3.2 Root cause analysis

The `claude-agent-sdk` doesn't talk HTTP-Anthropic directly — it spawns the bundled **Claude Code CLI** (Node) and speaks the CLI's **session protocol** over stdio. The CLI handshakes with the upstream and uses Anthropic's session/tool-use protocol on top. baidu-int relay implements the **Anthropic chat-completions HTTP shim** but not the **Claude Code session protocol**. Result: CLI thinks auth failed, returns the SDK's documented "synthetic" stand-in `AssistantMessage`, then both sides desync and the read-loop blocks.

This matches the **R69 spike finding §3.4**: "claude-agent-sdk depends on Claude Code CLI session protocol; non-Anthropic relays implementing only chat-completions will not work."

### §3.3 Why we didn't try harder

Per CONTEXT §6 R71 hard-red-lines:
- ❌ no autonomous install of Node `claude` CLI from third-party sources.
- ❌ no probing external Anthropic paid endpoints — only baidu-int relay authorized.

Both routes that could have produced a real trace are explicitly out-of-scope. R71's Option B branch is the correct response.

### §3.4 Path forward (R72+)

Three orthogonal unblock options, none autonomous:
- **(a)** User authorizes a real Anthropic API key + sets `CHRONOS_LIVE_ANTHROPIC=1` cron-side env. Probe + live-smoke run green next slot.
- **(b)** baidu-int operator extends relay to handle Claude Code session protocol (low probability, requires upstream cooperation).
- **(c)** Spike a **mock/replay seam** in the probe — record an SDK session against a real endpoint once, replay bytes through a stub `claude` CLI in CI. Larger scope, defers real-traffic confidence to whenever (a) lands.

R72 default plan now: pursue (c) as a *parallel* track (no auth required) while (a) waits on user. ADR-027 candidate to be drafted R72.
