### D2 — Why Anthropic Agents SDK over OpenAI Agents SDK

Three-way tie-break (both pass ADR-016 mapping cleanliness, both have rising ecosystem):

1. **Strategic fit**: MCP-native shops are underserved by existing observability tools (Langfuse/Phoenix lack MCP-aware semantics). OpenAI Agents SDK's competitive field (OpenAI's own tracing) is more crowded.
2. **First-user appeal**: Chronos's "agent pdb + git" framing fits MCP's tool-call interception point more naturally than OpenAI's internal Responses API.
3. **Leverage tail**: MCP standardisation (2024-Q4) is still compounding through 2026-H2. OpenAI Agents SDK peaks narrower.

The pre-authorised fallback clause hedges the one real risk (MCP fork-lifecycle) without forcing a round-level re-plan.

### D3 — Why 5 rounds for Arc B slice 1 (not 3)

CrewAI Arc B precedent was R49-R55 (7 rounds, stretched). N-run compare was R58-R60 (3 rounds, tight). Arc B slice 1 targets **5 rounds** because live-smoke gating requires two dedicated rounds (record + fork) whereas pure-function Arc A slices didn't. Bundle shape:

### F2 — Pre-authorised fallback clause as drift-prevention pattern (R68 新)

R26-era lesson: an ADR that commits to only one path without backup forces a round-level re-plan when the path blocks. Cost: 10+ rounds of drift (R26 adapter interface stall).

R68 innovation: ADR-026 §Fallback explicitly lists (framework, criteria, swap scope). If primary blocks at risks-spike stage, swap is pre-authorised — no re-ADR, no replanning detour. Criteria are hardened (3 must-all-hold) to prevent trivial fallbacks.

Candidate invariant: **"Every scope-commit ADR should name one fallback with gate criteria."** Pending second confirmation in a future scope ADR (ADR-027?).

### F3 — Arc B has a 5-round slice shape vs Arc A's 3-round (R68 prediction)

Arc A slice 4 was R62/R63/R64 (3 rounds: core/surface/proof+release). Arc B slice 1 pre-budgets R70/R71/R72/R73/R74 (5 rounds) because adapter shipping requires **live-smoke gating** (record + fork), not just proof. This is the same stretch CrewAI Arc B had (R49-R55, 7 rounds, further stretched by adapter-interface discovery).

Refinement to R60 bundle-shape invariant: "Arc slice = core + surface + proof = 1 minor version" is **Arc A specific**. **Arc B adapter slice = scaffold + live-smoke + alpha + fork + GA = 5 rounds = 1 minor version.** Two bundle-shape families co-exist. Pending R74 ship to confirm.
