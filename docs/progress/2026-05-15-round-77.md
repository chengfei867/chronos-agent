## §0 — Cover

R77 extends the R76 single-block `state_after['tool_use_id']` contract
(§5.1) to the multi-block case (§5.1.1). When an `AssistantMessage`
carries >1 `ToolUseBlock` (batched parallel tool dispatch) or a
`UserMessage` carries >1 `ToolResultBlock`, the recorder now surfaces
an ordered `state_after['tool_use_ids']` (plural) list. Mutual
exclusivity is binding: singular and plural fields MUST NEVER coexist
on the same Node. Slice-3 SQL consumers JOIN 1:N via
`json_each(state_after->>'tool_use_ids')`. No schema change — just a
contract pin extending §5.1's JSON-bag stamping. Three new unit tests
pin the contract; the adapter-1-3 zero-regression streak holds at
**R52→R77 = 25 rounds**.

## §2 — Work done

### §2.1 Recorder patch (`src/chronos/adapters/anthropic_agents/recorder.py`)

Added two symmetric `elif len(...) > 1:` branches inside `_translate()`:

- **Assistant side** (`AssistantMessage` with >1 `ToolUseBlock`):
  comprehension over `tool_blocks` filtering by
  `isinstance(getattr(b, "id", None), str) and b.id`, attribute-access
  on the truthiness check (B009-clean). If non-empty list survives the
  guard, `state["tool_use_ids"] = ids` (no singular `tool_use_id` set).
- **User side** (`UserMessage` with >1 `ToolResultBlock`): same shape,
  reading `b.tool_use_id` instead of `b.id`.

Order is preserved (no `sorted()`, no `set()`) so consumers can pair
`use[i] ↔ result[i]` by index. Mutual-exclusivity guarantee is
structural: the `len == 1` and `len > 1` branches are mutually
unreachable per Node.

Net diff: ~16 lines added, recorder imports / module API untouched.

### §2.2 Unit tests (`tests/unit/test_adapter_anthropic_agents.py`)

New §6.2.1 section pinning the contract (3 tests, all hermetic):

1. `test_record_multi_tool_use_block_persists_ids` — `AssistantMessage`
   with 3 `ToolUseBlock`s. Asserts ordered list match
   (`asst.state_after["tool_use_ids"] == ids`) and singular field
   absent.
2. `test_record_multi_tool_result_block_persists_ids` — full round-trip
   (Assistant 2-use + User 2-result). Asserts both Nodes carry
   byte-identical plural lists (the JOIN keyset) and neither carries
   singular.
3. `test_record_mixed_count_keeps_singular_and_plural_separate` —
   regression guard. Stream interleaves a single-block tool round-trip
   with a 2-block batched-use. Asserts the singular Node has only
   `tool_use_id`, plural Node has only `tool_use_ids`, with no
   cross-talk. Future refactor that collapses the two fields fails
   loudly.

Tests use the existing `_blk` / `_msg` / `_aiter` helpers and the
`@dataclass`-based `_StubBlockBase` (R76's `id` slot already in place).
Disambiguation in the mixed-count test goes by which key is present in
`state_after` (not by node ordering) — robust against any future
node-ordering change.
