## 2. R55 P0 — `tests/live/test_crewai_smoke.py`

### 2.1 Design

Modelled directly on `tests/live/test_real_llm_smoke.py` (R37-A1 + R48-A pattern), which already hosts a LangGraph live smoke and an AutoGen live smoke. Shape:

1. Module-level `pytestmark` list with three `skipif`:
   - `CHRONOS_LIVE != "1"` (opt-in gate, matches existing live tests)
   - `OPENAI_API_KEY` missing (API key guard)
   - `crewai` not importable (optional dep guard — CrewAI is in `[project.optional-dependencies].crewai`, not core)
2. `sqlite_store` fixture using `tmp_path / "crewai_smoke.db"` (isolated per run; spike13 used `tempfile.mkdtemp` which is fine for a one-shot script but pytest's `tmp_path` is idiomatic).
3. Single test function `test_crewai_tool_effects_smoke(sqlite_store, tmp_path)` that does, in order:
   - Builds the three `@tool("fetch_weather_api" / "read_file" / "query_db")` decorated functions inside the test (same stubs as spike13).
   - Creates `LLM(provider="openai", model=GLM-5, ...)` — the R54 CrewAI + OneAPI recipe (explicit `provider="openai"` bypasses LiteLLM native-constants validation).
   - Builds the investigator / summarizer agents, the two sequential tasks, the `Crew(Process.sequential)`.
   - Snapshots `id(crew)` pre-record (F5) and `crewai_event_bus._handlers` key list pre-record (F1).
   - Enters `rec.record(crew, thread_id=..., ...)` CM, calls `crew.kickoff(inputs={})`, calls `ref.submit_result(result)`.
   - Post-CM: F1 handler-keys snapshot compared to pre; F5 `id(crew)` compared.
   - F2 asserts `len(nodes) >= 4` (spike13 observed 13; `>= 4` is a safe floor that tolerates LLM path variation).
   - F3 iterates ToolUsage* nodes and asserts the expected effect tag for each known `tool_name`.
   - F4 soft-skips if no LLMCallCompletedEvent nodes exist OR if all usages are None (ADR-021 §D7 tolerance). If at least one usage is populated, assert `prompt_tokens >= 0` and `completion_tokens >= 0` for each — since CrewAI's `Usage` proto can legally be 0 on some channels, a strict `> 0` would be over-specification.
   - Closes `sqlite_store` (SQLite flush), then spawns `.venv/bin/chronos runs list --db <path>` and `.venv/bin/chronos runs show <run_id> --db <path>` subprocesses (F6), asserts exit 0 on both and that the `run_id[:8]` prefix appears in the `list` stdout.

### 2.2 Spec vs spike13 deltas

| spike13 (standalone, stdout-report) | test_crewai_smoke (pytest) |
|---|---|
| `print("[OK] ...")` / `print("[F?  ❌] ...")` | `assert ...` / `pytest.skip(...)` |
| `Path(tempfile.mkdtemp(prefix="spike13_")) / "spike13.db"` | `tmp_path / "crewai_smoke.db"` |
| F4 prints note even when all usage is None | F4 `pytest.skip(...)` when all usage is None |
| F3 unknown tool → `print("[F3 ??]")` | F3 unknown tool → silently skip (not a contract violation per R54 finding: LLMs pick 1/3 tools per run) |
| Module-level `from crewai ... import` | Imports inside the test function (so `@pytest.mark.skipif(not crewai_importable)` can actually skip instead of erroring at collection) |
| Exit code returned from `main()` | Test fails on any assertion, skips on any guard |

No logic deletions: every F1-F6 check survives the port. The only "loosened" branch is F4 (was "print and continue" in the spike; now `pytest.skip` with a self-explanatory reason because the ADR-021 §D7 tolerance would otherwise become an assertion hang).
