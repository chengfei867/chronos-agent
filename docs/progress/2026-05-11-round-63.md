### 3. Fix for the one failing test (slot-2 actual work begins here)

```
FAILED tests/unit/test_cli_compare.py::test_compare_auto_pivot_default_matrix_is_truncated
```

**Root cause**: the truncation-hint suffix `(showing 3 of 6 — pass --show-matrix for full)` was embedded **inside the `Table(title=...)` argument** in `_render_distance_matrix`. `CliRunner()` with no env override uses a ~80-col terminal; rich truncates long table titles with an ellipsis rather than wrapping. Result: the full title `"Pairwise distance matrix (metric v1) (showing 3 of 6 — pass --show-matrix for full)"` got rendered as `"Pairwise distance matrix (metric v1)..."` — so the assertion `assert "showing 3 of 6" in out` legitimately failed.

**Fix** (`src/chronos/cli/compare.py` `_render_distance_matrix`):

- Moved the trailer out of `Table(title=...)`.
- Table title is now just `"Pairwise distance matrix (metric v{version})"`.
- When truncated, emit the trailer as a separate `console.print(f"[dim](showing 3 of {total_pairs} pairs — pass --show-matrix for full)[/]")` line **after** the table. Rich doesn't truncate this at 80 cols — the body wraps or dim-styled text flows naturally.
- Added an inline comment referencing R63 so a future maintainer understands why the hint isn't in the title.

**Secondary benefit**: the trailer is now accessible to users as a real output line, not a truncation-prone title suffix — same UX even at 120+ cols.

## R63 Key Discoveries (for §5 wall)

- **Rich Table title ≠ safe place for truncation hints (R63 new)**. `Table(title=...)` gets ellipsis-truncated at narrow terminal widths; long titles lose their suffix silently. Rule: put **metadata** (metric version, counts) in the title; put **user-action hints** (`pass --show-matrix for full`) on a separate `console.print()` line. General lesson: anything that carries actionable next-steps for the user should not be rendered as decoration on another widget. ← **new**
- **A2 inheritance now six-in-a-row (R63 new, sixth verification)**: R48-A → R51 → R52 → R53 → R59 → R63. Every "land ADR + scaffold + tests" slot in the post-R47 era has produced a slot-1 / slot-2 split. The pattern is a structural constant, not a budget-fitting problem. R64+ should pre-budget two slots for any Arc-slice impl round, three slots if it also cuts a release. ← **refinement of R59 five-in-a-row**
- **"Different endpoint on same DB ≠ fixture piggyback" (R63 refinement of R59)**. The R59 "new fixture new scenario" rule was about not mutating a fixture to accommodate a new behavior. Reusing a fixture **unchanged** to exercise a parallel endpoint against the same DB does not violate the rule. Confirmed with `/runs/compare/auto` tests sharing `compare_n_scenario` + `compare_n_client` verbatim. ← **refinement**
- **Fourth layer of N=2 cross-layer guard (R63 new, R60 "bundle = 1 minor version" pattern ripening)**. The R58 N=2-equals-legacy guard now has four attestations: pure (R58) / CLI (R59) / HTTP-compare-n (R59) / HTTP-auto (R63). Every layer asserts the same numeric equivalence with slightly different plumbing. This is now a regression firewall: any attempt to optimize `merge_pivot_reports` or `auto_pivot_compare` that silently changes the summary numbers trips four tests simultaneously. Promote to "frozen-contract N=2 quadruple guard" in CONTEXT invariants. ← **new**
- **Pre-existing-this-slot test failure is still your problem (R63 new)**. Slot-1 left the matrix-truncation test failing. Even though slot-1 wrote the test, the "pre-existing" framing doesn't apply (R62 didn't ship with that test). The R62 click-8.3 case was different: *that* failure was in untouched HEAD. For inherited-WIP failures the correct framing is "this round's test" and the fix must land in this round's commit. Do NOT write a "pre-existing" disclaimer for failures you inherited from a slot-1 you can't directly attribute. ← **new**
