## Decisions

- **Treated prior-slot WIP as inheritance, not rework.** The code was ~1138 LOC of new work plus a surgical server.py extraction. Re-implementing would have been wasteful; the cron-slot-handoff-recovery skill explicitly prefers close-out-in-place when gates pass. After two small ruff fixes and format normalisation, the WIP is production-grade.
- **`uv.lock` restored to baseline** (chronos-agent@0.4.0a2 stale). The R65 lesson stands: transitive-dep churn is noise; every prior release (v0.5.0, v0.5.1) also shipped with the stale root version in `uv.lock`. Consistency matters more than freshness here — the real installed version comes from `pyproject.toml` + the editable install, not the lock. `git checkout -- uv.lock` before staging.
- **Bundle shape: three-round release.** v0.6.0 absorbs R65 (slice 5 surface, no tag cut R65 round) + R66 (audit + ADR-025 Draft, md-only no tag cut) + R67 (slice 5 closeout + tag cut). Different from the R58→R60 / R62→R64 pattern (core → surface → proof — each round substantive code) — this one is planning + surface + closeout. Still validates the invariant "Arc track closes with a minor version." Good variation to have on record.
- **core/tree.py sibling-module extraction.** Keeping `server.py._assemble_tree` / `_assemble_tree_with_descendants` as thin re-exports of `chronos.core.tree.assemble_tree` / `assemble_tree_with_descendants` preserves backward compatibility for any existing test or caller that imports from `server`. CLI gets a clean dependency: `cli/tree.py` → `core/tree.py`, never touching FastAPI's import graph. This is the same decomposition pattern used for `core/diff.py`, `core/auto_pivot.py` — Phase 4 has been a steady march of pulling "ex-server.py compute" into the core namespace.
- **ADR-025 Draft → Accepted in place (R57 invariant).** No new ADR cut; the footer updated to reflect R67 closeout evidence (CLI + tests + dogfood). Acceptance criteria stated in R66's Draft are all met.
