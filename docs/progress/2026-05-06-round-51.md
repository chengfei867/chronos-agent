### 3.2 `ruff format` on committed code is fine post-hoc

The prior slot ran the spike but didn't run `ruff format` before
commit. Today's slot can still apply the formatter retroactively —
it's cosmetic, doesn't change F-findings, and rolls in with the
ADR commit under a clear \"format nit\" bullet. Pattern is safe for
any spike or script file. **Would NOT do this for `src/` code
retroactively** because runtime-adjacent refactors need their own
commit for bisect hygiene.

### 3.3 CliRunner stderr capture regression (R52 blocker)

`click>=8.2` / `typer>=0.22` changed the default `CliRunner(mix_stderr)`
behaviour — stderr now always goes into `result.output` combined with
stdout unless you pass `mix_stderr=False`. That makes accessing
`result.stderr` raise `ValueError: stderr not separately captured`.
Our test suite (`tests/unit/test_cli.py`) uses the combined-read
pattern on exactly the affected code paths. Two clean fixes:

1. **Easy**: replace the module-level `runner = CliRunner()` with
   `runner = CliRunner(mix_stderr=False)` at
   `tests/unit/test_cli.py:~1`. Expected to green all 7 failures.
2. **Safer long-term**: pin `click>=8.1,<8.2` and `typer>=0.12,<0.22`
   in `pyproject.toml` until we decide to migrate.

R52 should go with option 1 — pinning ops-level tools fixes the
symptom but invites rot. Migrating to `mix_stderr=False` is strictly
a better test-fixture, matches what click's own docs recommend as of
8.2.

### 3.4 `cron-slot-handoff-recovery` skill — keeping as-is

The skill as-of today covers Option A2 (inherited green WIP, just
ship the docs) precisely. Applied straight from its playbook with
zero deviation. If a future slot finds a gap, they can patch it —
right now the skill is earning its keep.
