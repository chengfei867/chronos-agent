## §2 What shipped this round

### §2.1 New package `src/chronos/adapters/anthropic_agents/`

- **`_probe.py` (59 LOC)** — `HAS_CLAUDE_SDK: bool` + `CLAUDE_SDK_IMPORT_ERROR: ImportError | None` + `install_hint() -> str`. Mirrors CrewAI / AutoGen probe shape verbatim.
- **`recorder.py` (552 LOC)** — `AnthropicAgentsRecorder` implementing `RecorderProtocol`. Key design:
  - **Seam**: async iterator of `Message` objects. Works against both `ClaudeSDKClient.receive_messages()` (stateful client) and the top-level `query(prompt, options)` generator (stateless). `_resolve_iterator()` duck-types both.
  - **Dispatch**: class-name strings (`UserMessage` / `AssistantMessage` / `SystemMessage` / `ResultMessage`). Mirrors ADR-021 CrewAI event-class-name dispatch — recorder does NOT import `claude_agent_sdk` classes at runtime.
  - **Content summariser**: handles string, `None`, and block lists (`TextBlock` / `ToolUseBlock` / `ToolResultBlock` / `ThinkingBlock`) — the four block types enumerated in R69 spike #2.
  - **Usage projection**: Anthropic `input_tokens` / `output_tokens` / `cache_creation_input_tokens` / `cache_read_input_tokens` → Chronos `Usage`. Cache tokens summed. All-zero → `None`. Type drift → `AdapterError` (ADR-021 invariant: never silent).
  - **Tool surfacing**: `AssistantMessage` + single `ToolUseBlock` sets `tool_name` / `tool_input`; `UserMessage` + `ToolResultBlock` sets `tool_output`; `is_error=True` sets `error_message`.
  - **Node name**: message class → base name (`user` / `assistant` / `system` / `result`); if assistant + single ToolUseBlock, append `:<tool_name>` (ADR-020 three-segment pattern's first-segment analogue for single-tool calls).
  - **`fork()` stub**: `raise NotImplementedError("R73: delegate to `claude_agent_sdk.fork_session()`")`. R73 will swap in the delegate.
- **`__init__.py` (158 LOC)** — public exports + extensive module docstring with usage pattern, R70 scope, R70 non-goals, R71/R72/R73/R74 forward pointers. `anthropic_agents_adapter` = module-level `AdapterProtocol` instance for R32-B dynamic registry enumeration.

### §2.2 Tests `tests/unit/test_adapter_anthropic_agents.py` (577 LOC, 34 tests)

Covers structural conformance, usage projection (6 cases), content summariser (6 cases), node-name dispatch (5 cases), translate dispatch (8 cases), record happy + failure + ClaudeSDKClient-style + non-iterable + SDK-drift, fork-stub, factory channels, probe, kind-map override.

Critically: **every test uses in-memory async-generator duck runtimes**. No `claude_agent_sdk` import required to run the suite. The recorder has no runtime dependency on the SDK — only the `anthropic_agents` extra install adds it.

## §4 Findings for the wall

1. **A2 inheritance chain 九连, first Arc B entry**: R48-A → R51 → R52 → R53 → R59 → R63 → R65 → R67 → **R70**. All nine share the same shape: prior slot ships substantial scaffold + tests (green or almost-green) but runs out of budget before close-out (CHANGELOG + progress doc + CONTEXT refresh + commit + push). **R63's pre-budget rule "2 slots per impl round" continues to hold**; R70 confirmed. Arc B's first entry conforming with zero deviation from Arc A pattern is itself validating — the structural constant of autonomous cron scheduling isn't tied to feature area.

2. **Class-name dispatch pattern 三连 (R70 new)**: CrewAI adapter (ADR-021) / AutoGen adapter (ADR-020 three-segment node names) / **Anthropic Agents adapter (R70)** all use `type(msg).__name__` string dispatch rather than isinstance checks against SDK classes. Rationale: recorder module is SDK-import-free at runtime (probe-gated), so isinstance is impossible. Class-name string is the duck-typed replacement. Promote to **candidate invariant**: "SDK-optional-dep adapters dispatch by runtime class name, not isinstance". Pending fourth confirmation (Arc B slice 2 adapter if R74+ lands one).

3. **Four-block message content invariant (R70 new, R69 spike #2 confirmation)**: Anthropic `Message.content` is either `str | None` (simple text case) or `list[TextBlock | ToolUseBlock | ToolResultBlock | ThinkingBlock]` (rich case). No fifth block type observed across R69 source inspection. Summariser handles all four + `None` + `str`. If a new block type lands in a future SDK release, ADR-015 extractor contract v2 may need amendment — but R70 codebase fails loud rather than silent-lossy (unknown blocks → `_summarise_block` fallback to class-name tag). Worth grep-checking in R71 live smoke.

4. **Pre-1.0 pin ceiling is library-maturity-aware (R69 promoted, R70 applied)**: `claude-agent-sdk>=0.1.80,<1.0` is the first Chronos extra to use a next-major ceiling. ADR-022's next-minor ceiling stays for 1.x stable deps (CrewAI `<2.0`, langgraph `<2.0`, etc.). The rule, now codified in pyproject comment + ADR-026 §7: "**next-major ceiling for 0.x alpha libraries with additive-only patch cadence; next-minor ceiling for 1.x stable SemVer**."

5. **Tests run without the optional extra installed (R70 new, reinforces ADR-016 optional-dep precedent)**: `test_adapter_anthropic_agents.py` does NOT `skipif not HAS_CLAUDE_SDK`; instead it uses duck-typed async-generator runtimes with class-name dispatch. This is **stricter than** the CrewAI adapter's `skipif not HAS_CREWAI` pattern, and it's possible because the recorder never imports SDK classes at runtime (class-name dispatch + duck-typed content blocks). **Rule refinement**: when an adapter's recorder has zero runtime SDK imports (probe-gated only), the unit-test suite should run unconditionally rather than skip — catches structural regressions even in minimal dev environments. Pattern to generalise to Arc B slice 2 if applicable.

6. **uv.lock regeneration was real this time (R70 new, context to lockfile-trap rule)**: `git status` showed `M uv.lock` with 2541 diff lines. Per the skill's lockfile-regeneration trap recipe, checked `git diff pyproject.toml` first — the diff was **non-empty** (new `anthropic_agents` optional extra). This means the lockfile churn is **real and required**, not uv-version noise. Unlike R65 / R68 where pyproject was untouched. Committed as-is. **Refinement to `cron-slot-handoff-recovery` skill**: the trap triggers *only* when pyproject is untouched; real manifest changes carry real lockfile diffs. The 2-tool-call recipe correctly handled both cases in R70.
