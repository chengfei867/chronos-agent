### 1.3 Spike #1 — MCP fork-lifecycle (was blocker: Policy A fresh-server vs Policy B session-reseed)

- Inspected `src/claude_agent_sdk/__init__.py` → `fork_session` is **first-class exported**.
- Read `_internal/session_mutations.py:230-410`: signature `fork_session(session_id, up_to_message_id=...)`; implementation is pure transcript-JSONL rewrite — assigns new UUIDs, truncates history at the requested message, writes a fresh session file. There is also a `fork_session_via_store(...)` variant for the SessionStore adapter shipped per TS-SDK parity (CHANGELOG ref #837).
- `grep -n "mcp" _internal/session_mutations.py` → **0 hits**. Confirms fork has zero coupling to MCP server state.
- Architectural confirmation: MCP servers are per-CLI-subprocess. When a forked session is resumed, the subprocess reconnects stateless; the transcript carries no MCP session identity beyond the tool-use/tool-result message blocks themselves.
- **Resolution**: Neither Policy A nor Policy B. chronos-agent's fork primitive **delegates to the SDK's native `fork_session()`**. No custom re-seed logic, no session-lifecycle policy to implement. ADR-026 §Fallback remains dormant — not triggered.

### 1.4 Spike #2 — Recorder entry point (was design: `agent.iter` vs `agent.stream`)

- Grepped the entire SDK for `iter_`, `agent.iter`, `agent.stream` → only internal `__aiter__` usages in `_internal/query.py`. The names in ADR-026 §5 were speculative and **do not exist** in the shipping API.
- Actual recorder-relevant primitives:
  - `query(prompt, options) -> AsyncIterator[Message]` — stateless fire-and-forget async generator.
  - `ClaudeSDKClient` — stateful async context manager. Usage pattern confirmed in `examples/streaming_mode.py:64-70`:
    ```python
    async with ClaudeSDKClient(options=...) as client:
        await client.query("...")
        async for msg in client.receive_response():
            ...
    ```
  - Message union: `UserMessage | AssistantMessage | SystemMessage | ResultMessage`.
  - Content-block types: `TextBlock | ToolUseBlock | ToolResultBlock | ThinkingBlock`.
- **Resolution**: Recorder seam = wrap the `receive_response()` async iterator (for stateful multi-turn) and the `query()` generator (for one-shot). This is the third verification of the "stream → log" pattern after LangGraph callbacks and CrewAI event bus. Maps cleanly onto `RecorderProtocol` — no ADR-016 amendments needed.

## 2. Decisions and rationale

### D1 — Delegate fork to SDK primitive, don't custom-implement

**Decision**: chronos-agent's Anthropic Agents SDK adapter wraps `fork_session()` rather than implementing its own re-seed + re-invoke pattern (as LangGraph and CrewAI adapters do).

**Rationale**: The SDK maintainers have already solved transcript truncation correctly, including UUID assignment and JSONL atomicity. Reinventing this is pure risk without upside. The `AdapterProtocol.fork()` contract is framework-agnostic; it does not constrain the implementation strategy. LangGraph's `update_state + invoke(None)` and CrewAI's plan-artifact replay are implementation details of those adapters, not ADR-016 requirements.

**Consequence**: The R73 fork-primitive round will be **smaller than originally budgeted** — likely half a day of adapter code plus conformance + live-smoke tests, no new plan-artifact JSON, no custom policy. This is a net win on R74 burn-down.

### D2 — Pin ceiling at `<1.0` not `<0.2`

**Decision**: Upper bound `<1.0` rather than `<0.2` per [ADR-022] precedent.

**Rationale**: [ADR-022] (CrewAI bump) used a next-minor ceiling because CrewAI was already 1.x with stable SemVer. `claude-agent-sdk` is pre-1.0 alpha with 83 releases in the 0.1.x line and observed additive-only 0.1.x bumps. A `<0.2` ceiling would force a bump-round every time Anthropic cut a patch, with no corresponding breakage risk — pure churn. `<1.0` gives us the natural pre-1.0 compatibility envelope and shifts the stability decision to the R74 retro (by which point 0.1.90+ or 0.2 will clarify direction). This is consistent with how PyPI's broader Python ecosystem treats pre-1.0 SDKs.

**Consequence**: If 0.2 ships breaking changes before R74, we'll catch it via live-smoke on R71+ and tighten in-round. The cost of a reactive tightening is one patch commit; the cost of proactive tightening is N bump-rounds. Net expected value favours the loose ceiling.

### D3 — In-place promotion interprets "scope decided" not "gates closed"

**Decision**: Draft→Accepted flip happens when the **Arc B scope question** is settled (i.e. what framework, with what primitives, at what pin). Release gates AC-1..AC-5 are separate; they close at R74 GA via a commit note, not a second status flip.

**Rationale**: R57 invariant says ADR promotion is in-place. But "ADR accepted" has two possible meanings: (a) the design decision is made, (b) the implementation has shipped. Conflating them forces every scope-type ADR to remain Draft until release, which loses the informational value of the Accepted label. For **scope** ADRs (this one, [ADR-023], [ADR-026]), Accepted = scope decided; for **interface** ADRs ([ADR-016]), Accepted = contract frozen and one conforming adapter green. The §Acceptance-criteria block now makes this explicit: AC-1..AC-5 are tracked through release but do not gate the status field.

**Consequence**: R74 release commit will say "ADR-026 acceptance gates AC-1..AC-5 closed" rather than "ADR-026 Draft→Accepted" — which matches reality: the scope flip happened now.

## 5. Findings (for CONTEXT §5 wall)

1. **SDK-native `fork_session` removes the custom-fork design surface.** Arc B slice 1's fork primitive becomes a one-function wrapper, not a mini-subsystem. R73 fork-round budget shrinks meaningfully.
2. **Recorder seam is `ClaudeSDKClient.receive_response()` async iterator.** Stream-to-log pattern verified across all four adapters (LangGraph callbacks, AutoGen sync-wrap+log, CrewAI event bus, Anthropic SDK async stream). ADR-016 contract stable through a fourth framework without amendments.
3. **Pin `claude-agent-sdk>=0.1.80,<1.0` with pre-1.0 compatibility rationale.** Pin policy differs from [ADR-022] because target library maturity differs (pre-1.0 alpha vs 1.x stable). ADR-022 precedent applies *mechanically* to 1.x libraries only.
4. **Live-smoke needs Node.js installed on CI runner** (R71 infrastructure note). `HAS_CLAUDE_CODE` probe must check both Python import and CLI subprocess resolvability.
5. **Fallback clause never triggered.** First Arc B round-1 contingency spec whose pre-authorised fallback path (OpenAI Agents SDK swap) was *not* needed — all primary-path risks dissolved on inspection. Precedent: when we write a Draft ADR with a pre-auth fallback, spending one round on a source-inspection spike before committing to the fallback is cheap insurance.
