## Goal

Land PH3-02 — the first Phase 3 on-ramp brick. Adapters annotate each node with an `effects` list, and the Web UI warns users before they fork a node that touched the real world. Per ADR-019 (R43-B) Chronos stays **explicitly non-sandboxed** — warnings are the honest compromise, not fake safety.

## Deliverables

### 1. Adapter heuristic — `src/chronos/adapters/effects.py` (170+ lines)

`classify_effects(kind: NodeKind, node_name: str, override: list[str] | None = None) -> list[str]`:

- `NodeKind.LLM` → `["llm"]`
- Regex on `node_name` (snake_case-aware):
  - `network` — http/api/fetch/request/get/post/download/upload
  - `fs` — read/write/file/path/save/load
  - `db` — db/postgres/redis/sql
  - `external` — send_slack/send_email/run_shell/subprocess

`DANGEROUS_EFFECTS_DEFAULT = {"network", "fs", "db", "external"}` — **`llm` deliberately excluded** because forking is precisely about re-running LLM reasoning; flagging it would spam warnings on every interesting fork.

`count_dangerous_downstream(store, run_id, from_step)` — helper for the fork-plan preview (counts how many dangerous nodes will be re-executed if you fork at `from_step`).

### 2. Adapter integration

Both `LangGraphRecorder.__init__` and AutoGen `Recorder.__init__` now accept `effects_map: dict[str, list[str]] | None = None` (per-node-name override for power users). Every recorded `Node` gets `metadata["effects"] = classify_effects(kind, node_name, override=effects_map.get(node_name))`.

**Zero SQL migration**: `metadata_json` column has existed since v0.1.0; `Node.model_dump()` auto-exposes `metadata.effects` through the API.

### 3. UI effects badge + Fork warning — `frontend/src/components/NodeDetails.tsx`

- **Effects row** under the identity table: `<Tag color=...>` per effect. Color map: `llm=purple`, `network=orange`, `fs=gold`, `db=volcano`, `external=red`.
- **Amber `<Alert warning>` banner** at the top of the drawer when `hasDangerous`. Title: "This node may have side effects". Body explains forking re-runs the real-world operation (re-charge, re-send, re-write) and suggests forking from a pure node for reasoning-only exploration.
- **Danger-styled Fork button** when `onFork` prop is provided AND node is dangerous: AntD `danger` styling + ` · re-triggers side effects` appended to the label.

## Key lessons

1. **Python regex `\b` word-boundary trap**: `_` is a word character in Python regex, so `\bhttp\b` does **not** match `http_get`. For snake_case compound names use `\bhttp(_\w+)?\b` or the looser `\b\w*http\w*\b`. Saved as skill `python-regex-word-boundary-underscore-trap`.
2. **`SqliteStore` construction**: class factory is `SqliteStore.open(path)`, not `SqliteStore(Path(path))` directly. The raw `__init__` takes a `sqlite3.Connection`.
3. **`Run.adapter_thread_id` not `thread_id`**: the field is namespaced to distinguish from Chronos's own thread concept.

## Decisions

- **Option B (metadata-only) beats Option A (first-class column)** — zero SQL migration, API/UI transparent. R43-D recommendation validated.
- **LLM in DANGEROUS_EFFECTS_DEFAULT: no**. Forking to re-run LLM reasoning is the core happy path; warning would be noise.
- **Fork button danger styling is predicated, not default**: the v0.3.0 user-visible primary signal is the Alert banner. When future `onFork` integration happens, the button auto-activates.
