## 1. Why this round

CONTEXT.md §6 listed R48-B as "优先推荐" (0.5 round): before cutting
v0.4.0a2 I want the visual density of the fork-safety UX to actually
match the prose in `forking-safely.md`. Four dangerous-effect families
all reading as orange-ish text chips with subtle color hue differences
is readable on a 27" monitor but starts to blur at modal widths on a
13" laptop, and the `dangerous_samples` list in the fork modal was
rendering tags as a flat `"network, fs"` comma-joined text rather than
as chips at all.

Icons are a cheap, low-risk affordance: they raise the scan speed of a
tag on a crowded screen without changing any semantics, schema, or
test fixture.

## 2. What landed

### 2.1 `frontend/src/components/NodeDetails.tsx`

- Imported five lucide icons: `Brain, Globe, HardDrive, Database,
  ExternalLink`.
- New `EFFECT_ICONS` map (parallel to existing `EFFECT_COLORS`), typed
  as `Record<string, ComponentType<{size?: number | string}>>` so
  unknown tags cleanly fall through (`undefined` → no icon).
- Extracted `EffectTag` as a named export: a thin wrapper around AntD
  `Tag` that composes `EFFECT_COLORS[tag]` + `EFFECT_ICONS[tag]` + an
  optional `label` override. `display: inline-flex; gap: 4px` keeps the
  icon + text vertically centered.
- Replaced the inline `effects.map(tag => <Tag color={...}>...</Tag>)`
  loop with `<EffectTag tag={tag} label={t(...)} />`. Behavior
  identical except for the leading 12px icon.

### 2.2 `frontend/src/components/ForkPlanModal.tsx`

- Imported the new `EffectTag` from `./NodeDetails`.
- Tag-count histogram in the dangerous-effects Alert: now renders
  `<EffectTag tag={tag} label={`${tag}:${n}`} />` instead of
  `<Tag color="orange">{tag}:{n}</Tag>`. Biggest visible change:
  previously every tag was orange regardless of family, now each one
  wears its own color + icon.
- Dangerous-samples list: previously `<Text code>step 3</Text> name —
  <Text type="secondary">{tags.join(", ")}</Text>`. Now each tag in the
  row renders as its own chip. `Space size={2}` with `wrap` keeps the
  layout tidy when a sample has three tags on it.
- AntD `Tag` import kept in the ForkPlanModal file: the "forks at last
  node" geekblue chip higher up still uses raw `Tag` (not an effect
  tag), and there's no point forcing it through the new abstraction.

## 5. Risks / follow-ups

- **Icon font weight in dark mode** — lucide icons inherit `stroke`
  color from CSS. AntD colored Tags set their own text color
  contextually; the 12px icon size should read fine in both themes,
  but this deserves an eyeball pass on the screenshot refresh in
  R48-C.
- **`ExternalLink` icon semantic clash** — `ExternalLink` in lucide is
  the standard "opens in a new tab" arrow. Using it for the
  `external` effect family (meaning "external side effect like
  payment / email send") may be misread as "this opens a link". If
  users flag confusion during dogfood, swap to `Send` or
  `Zap`. Noted in follow-up but not escalated — `external` is the
  lowest-frequency effect family empirically.
- **SIM / visual-density vs. reading-speed** — I didn't run an A/B
  comparison. The change is defensible on aesthetics; if it turns out
  icons *slow* reading (they can if icon-text binding is weak), we
  can drop to icon-only with i18n tooltips. Keep this in the pocket
  for v0.4.0a2 dogfood feedback.
- **`EffectTag` export from `NodeDetails.tsx`** — the component is
  conceptually about effects, not about node-details specifically.
  Cleanest refactor is moving it to a new file `EffectTag.tsx` and
  re-exporting from `NodeDetails`. Didn't do that here to keep the
  diff small; if a third consumer appears in R49+, that's the moment
  to promote it.
