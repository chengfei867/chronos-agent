## §2 Work done

### §2.1 P0 probe — SDK API shape (refutation of R71 stub assumption)

R71 stub message claimed `fork_session` was an internal API requiring
hooks into `_internal.session_mutations`. Probe disproved this:

- `claude_agent_sdk.fork_session` **is a public top-level function**.
- Signature: `fork_session(session_id: str, *, up_to_message_id: str | None,
  title: str | None, directory: str | os.PathLike[str] | None) -> ForkSessionResult`
- Plus a sibling `fork_session_via_store(...)` for in-process `SessionStore`
  backends (kept in reserve for the relay-fallback path).
- `AssistantMessage` and `ResultMessage` already carry `.session_id` and
  `.uuid` fields — exactly the anchors a `fork()` implementation needs.

**Pre-existing surprise**: `record()` (R70) already stamps `state_after =
{"uuid": …, "session_id": …}` for assistant/result nodes. Zero schema
change needed — fork() just reads the anchor.

### §2.2 `AnthropicAgentsRecorder.fork()` implementation

**Path**: `src/chronos/adapters/anthropic_agents/recorder.py` (lines 605-770).

Algorithm:

1. Validate parent run + anchor node belong together; child thread_id
   differs from parent thread_id.
2. Read `parent_node.state_after["session_id"]` + `["uuid"]` — raise
   `AdapterError("no SDK session_id …")` if the anchor pre-dates the
   capture (e.g. SystemMessage init before any AssistantMessage).
3. Lazy-import `claude_agent_sdk.fork_session`; raise `AdapterError` if
   the optional extra is missing (mirrors the R70 import-error pattern).
4. Call `fork_session(parent_session_id, up_to_message_id=parent_uuid,
   title=task_description)` → child `session_id`.
5. Build a child shell `Run` (`adapter_thread_id=child_thread_id`,
   `parent_run_id=parent_run_id`, `RunStatus.RUNNING`).
6. Yield a `ForkRef` with two extension methods:
   `submit_runtime(runtime)` lets the caller hand in their resumed
   `ClaudeSDKClient`; `sdk_session_id` exposes the child id for
   `ClaudeAgentOptions(resume=…)`.
7. On `__exit__`: if a runtime was submitted, drain it via the same
   `_consume()` pipeline as `record()`. In-finally `RunStatus` stamping
   (`COMPLETED` on clean exit, `FAILED` on user exception).
8. Atomically write the `Fork` row inside the same transaction as the
   child Nodes (matches LangGraph adapter idiom; preserves
   parent→child→fork referential integrity even if the user-block
   raises).

### §2.3 Unit test layer (`tests/unit/test_adapter_anthropic_agents.py`)

R71's single negative test (`test_fork_raises_not_implemented`) replaced
with **5 positive + negative cases**:

| # | Test | Assertion |
|---|---|---|
| 1 | `test_fork_happy_path_persists_run_nodes_and_fork_row` | full happy path with monkey-patched `claude_agent_sdk.fork_session` returning a stub `session_id`; verifies SDK call site received correct args, `ForkRef` populated, `Fork` row links parent→child, child Run COMPLETED |
| 2 | `test_fork_rejects_unknown_parent_run` | `AdapterError` matches `"parent_run_id="` |
| 3 | `test_fork_rejects_node_from_different_run` | record two runs, fork run-A using run-B's node — `AdapterError("does not belong")` |
| 4 | `test_fork_rejects_same_thread_id` | `AdapterError("must differ from parent")` |
| 5 | `test_fork_rejects_anchor_without_session_id` | parent anchor is a SystemMessage with no session_id+uuid → `AdapterError("no SDK session_id")` |
| 6 | `test_fork_persists_failed_status_on_user_block_exception` | exception inside `with` block → child Run = FAILED, Fork row still written (atomic transaction) |

All duck-typed — no `claude_agent_sdk` install required at test time.
