#### 2.2.2 Lint fix — stale `# noqa: RUF001` on `"≠"`

`ruff check` flagged `RUF100` on line 55 of `src/chronos/cli/compare.py`: the `# noqa: RUF001` on `"changed": "≠"` is unused because ruff's RUF001 catches ambiguous *Latin-lookalike* characters (cyrillic e vs latin e, greek question mark vs semicolon, etc.) and `≠` (U+2260 NOT EQUAL TO) isn't on the list. U+2212 MINUS (`−`) *is* on the list — the `# noqa: RUF001` on line 57 for `"removed": "−"` stays. Dropped the `# noqa` from line 55, kept the comment as an inline note. This is a tiny variant of the R58 lessons about `# noqa: RUFxxx` directives (which codes cover which chars).

## 4. Key decisions / lessons (candidates for §5 invariants)

### 4.1 A2 inheritance is now 5-in-a-row (R48-A → R51 → R52 → R53 → R59)

This is the fifth consecutive major-feature round where slot 1 did the code + scaffold and slot 2 did the close-out (CHANGELOG + ADR / progress doc / CONTEXT / commit / push). The cron-slot-handoff-recovery skill's A2 shape explicitly anticipates this; the recipe ran verbatim with zero surprises. **I'm updating CONTEXT §5 to treat "A2 close-out is the post-implementation-slot archetype" as a soft invariant rather than a curiosity.**

### 4.2 "≠" is not a RUF001 char — only cross-script-lookalikes are

R58 lessons #4 noted `RUF043` quirks around regex metachars in `pytest.raises(match=...)`. R59 adds a sibling: `# noqa: RUF001` only applies to chars on ruff's ambiguous-latin table. `≠` (U+2260) isn't — the closest match is `=` (U+003D) from a completely different script. `−` (U+2212) *is* on the table (looks like `-` U+002D). Rule of thumb: before slapping `# noqa: RUF001` on a char, confirm it's actually flagged by ruff with that code. If ruff doesn't complain without the `# noqa`, the `# noqa` is dead code. Low-value but adding to the lessons table.

### 4.3 N=2 cross-layer regression guards are the right pattern

Three places now assert "N=2 via the new N-run path equals the 2-run path":

- `test_merge_pivot.py::test_summary_matches_diff_report_for_n2` (R58, pure-function layer)
- `test_cli_compare.py::test_compare_n2_summary_matches_chronos_diff_summary` (R59, CLI layer)
- `test_api_server.py::test_compare_n_n2_matches_compare_2run_summary` (R59, HTTP layer)

Three layers, three guards. If any future change to `merge_pivot_reports` subtly drifts from `DiffReport.summary` semantics, we catch it at exactly the right layer with a byte-level diff in the failure message. **This is the "frozen-contract triangulation" pattern**: the CLI-first-API-locked discipline (design doc §3.3) implicitly asks for it. Candidate skill after one more round confirms.

### 4.4 Fixture redundancy is cheap — don't share `compare_n_scenario` with `scenario`

The existing `scenario` fixture in `test_api_server.py` has only 2 runs. I could have extended it with a third run, but that would have bloated every unrelated test (healthz, list-runs, tree, replay, fork-plan, …) by 1 extra run it doesn't care about. New fixture = new scenario = zero cross-test side effects. Cost: ~80 LOC, one full seed function. Value: test isolation that will matter every time we touch this file. No regret.
